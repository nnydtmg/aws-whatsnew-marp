# AWS CloudWatch Alarm Mute Rules eliminate alert fatigue

**カテゴリ:** What's New
**公開日:** 2026-02-10
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-cloudwatch-alarm-muting-rules](https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-cloudwatch-alarm-muting-rules)

---

## 要約

AWS CloudWatchのアラームミュートルール機能により、顧客はデプロイメントやメンテナンス期間中にアラーム通知を一時的に消音でき、アラート疲労を軽減しながら監視可視性を維持することができるようになりました。

## 詳細

- AWS CloudWatchにアラームミュートルールという新機能が追加されました。この機能により、顧客は計画的なデプロイメント、メンテナンスウィンドウ、および業務外の時間帯にアラーム通知を一時的に消音することができます。
- アラームミュートルールは、1回限りまたは繰り返し設定が可能で、最大100個の個別アラームに対して通知を消音することができます。
- この機能により、アラート疲労を軽減しながら、インフラストラクチャ全体の監視可視性を維持することが可能になります。
- ミュートルール有効期限後は、アラームが同じ状態に留まっている限り、以前消音されたアクションが自動的にトリガーされるため、重大な問題が見落とされることはありません。
- スクリプトベースの回避策を忘れるという運用リスクが排除され、計画的なアクティビティ中のアラート通知ノイズが削減されます。
- CloudWatch Alarm Mute Rulesは、アラームレベルのミュート機能をサポートするすべてのAWSリージョンで本日から利用可能です。
- Amazon CloudWatchコンソールを通じてミュートルールを作成することができます。
