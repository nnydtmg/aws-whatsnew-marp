# Amazon Redshift
自動最適化機能の専用コンピュート割り当てに対応

**発表日: 2026年2月9日**

---

## 概要

### 3つの主要な改善

- **ユーザーワークロード分離**: 自動最適化操作を独立したコンピュートで実行可能
- **手動スケジュール不要**: 自動テーブル最適化（ATO）、ソート、バキューム、分析を自動実行
- **コスト制御**: リソース上限を設定して専用コンピュートの利用量を管理

---

## 前提・背景

### データウェアハウスの課題

- **パフォーマンス競合**: ユーザークエリと保守作業のリソース競争
- **手動スケジューリング**: オフピーク時に最適化操作を手動で計画する必要
- **DBA負荷**: 24時間体制でのメンテナンス作業管理

### Amazon Redshiftの進化

- 2025年: マルチクラスター環境でのautonomics対応
- 2026年: 自動最適化専用のコンピュート割り当て機能登場
- 継続的なカイゼン: エンタープライズワークロード対応を強化

---

<!-- _class: small -->

## 主な機能と改善

### 1. 自動最適化機能（Autonomics）

- **自動テーブル最適化（ATO）**: テーブルの物理配置を最適化
- **自動テーブルソート（ATS）**: ソートキー列のデータ順序を保持
- **自動バキューム**: 削除済み行領域をクリーンアップ
- **自動分析**: テーブル統計を自動更新

### 2. 専用コンピュート割り当て

- ユーザークエリと独立した計算リソース
- 自動最適化専用のクラスターノード確保
- パフォーマンス影響ゼロで24時間最適化実行

---

## パフォーマンスと効果

### コスト管理機能

| 項目 | 説明 |
|------|------|
| **リソース制限** | プロビジョニングクラスターに上限設定可能 |
| **クラスター効率** | 追加ノード購入せずに管理オーバーヘッド削減 |
| **自動バランシング** | システムが利用可能リソース内で最適実行 |

### 監視と可視化

- **SYS_AUTOMATIC_OPTIMIZATION テーブル**: 全autonomics操作の詳細ログ
- **プロビジョニング/サーバーレス両対応**: 統一されたモニタリング
- **使用量メトリクス**: 実コスト把握が容易に

## ユースケース

### 24時間稼働型ウェアハウス

```
実行パターン:
┌─────────────────────────────────┐
│ ビジネスアワーズ（ピーク）      │
│ → ユーザークエリ優先実行        │

│
│ スケーリング
│
┌─────────────────────────────────┐
│ 専用コンピュート割り当て        │
│ → autonomics常時実行可能        │
│ → パフォーマンス低下なし        │
└─────────────────────────────────┘
```

- 各顧客クラスターのメンテナンスを独立実行
- SLAコンプライアンスの確保
- 運用チームの手作業削減
