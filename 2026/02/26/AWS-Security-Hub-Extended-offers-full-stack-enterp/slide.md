# AWS Security Hub Extended

フルスタックエンタープライズセキュリティと厳選パートナーソリューション

**発表日: 2026年2月26日**

---

## 概要

### AWS Security Hub Extended の3つの主要特徴

- **フルスタックセキュリティ**: エンドポイント、アイデンティティ、メール、ネットワーク、データ、ブラウザ、クラウド、AI、セキュリティオペレーションなど複数のレイヤーをカバー

- **厳選パートナー統合**: 事前交渉された信頼できるセキュリティソリューションを一箇所から管理

- **購買・運用の簡素化**: 複数ベンダー交渉の排除、単一請求書、長期契約不要で導入が容易

---

## 前提・背景

### エンタープライズセキュリティの課題

- **複雑な調達プロセス**: 複数ベンダーとの個別交渉による時間・コスト増加
- **統合・運用の負担**: 複数ツールの運用により、SOC チームの負担が大きい
- **可視性の断裂**: 各ツールからのデータが分散し、統合分析が困難

### 市場動向

- エンタープライズは多層防御（Defense in Depth）の必要性を認識
- セキュリティツールの統合による運用効率化への需要が高まる
- 従量課金モデルへの移行による柔軟な調達スタイルの普及

## 購買・統合モデル

### AWS が販売者となることのメリット

- **事前交渉済み価格**: 従量課金制での割引を適用
- **単一請求書**: 複数パートナーソリューションを1つの AWS 請求に統合
- **長期契約不要**: 柔軟に購買・解約が可能
- **簡単なオンボーディング**: Security Hub コンソール内から直接デプロイ

### 統合・集約の仕組み

- **OCSF スキーマ**: すべてのパートナーソリューションからのセキュリティ検出結果を Open Cybersecurity Schema Framework（OCSF）スキーマで自動集約
- **統一コンソール**: Security Hub 内から直接パートナーソリューションにアクセス・管理可能
- **自動メータリング**: オンボーディング後、従量課金メータリングが自動実行され、月次請求に含まれる

## 利用方法・デプロイ

### ワークフロー

1. **Security Hub コンソールでパートナー確認**
   - 利用可能なセキュリティソリューションを確認

2. **パートナーソリューションの選択・デプロイ**
   - Security Hub 内からワンクリックでデプロイ開始
   - 各パートナーによるセットアップガイドを参照

3. **検出結果の集約・管理**
   - すべてのセキュリティ検出結果が OCSF スキーマ経由で Security Hub に集約
   - 統一ダッシュボードでリアルタイム監視

4. **請求・運用**
   - 月次従量課金で AWS 請求に自動統合
   - 必要に応じて柔軟に契約内容を変更

### 利用可能リージョン

- Security Hub が利用可能なすべての **AWS 商用リージョン**で一般提供
- 従量課金制または定額料金制から選択可能

## 効果・メリット

### 調達・運用コストの削減

- **ベンダー交渉の廃止**: AWS とのワンストップ交渉で調達時間を大幅削減
- **統一請求**: 複数請求書の管理が不要に
- **長期契約不要**: 従量課金制で必要に応じた柔軟な変更が可能

### セキュリティ運用の効率化

- **統一管理**: 複数パートナーソリューションを Security Hub から一元管理
- **検出結果の自動集約**: OCSF スキーマによる統一フォーマットで分析が容易
- **SOC 負担軽減**: ツール統合による運用オーバーヘッドの削減

### セキュリティ態勢の強化

- **フルスタックカバー**: エンドポイントからセキュリティオペレーションまで総合的に対応
- **脅威検知の向上**: 複数ベンダーの検知能力を組み合わせた多層的な防御
- **対応速度の向上**: 統一ダッシュボードからの迅速な脅威対応

## ユースケース

### 金融機関

- 複数の規制要件（コンプライアンス）に対応するための包括的なセキュリティソリューション
- 統一ダッシュボードで監査人への説明資料作成が容易に

### 大規模エンタープライズ

- 複数拠点・複数ツールの統合管理による SOC の効率化
- グローバルセキュリティポリシーの統一実装

### 成長企業

- セキュリティ人材が限定的な中での包括的なセキュリティ体制構築
- 従量課金制による初期投資最小化

## まとめ

### AWS Security Hub Extended の位置づけ

- エンタープライズのフルスタックセキュリティニーズを単一プラットフォームで実現
- 信頼できるパートナーソリューションを AWS が厳選・統合
- 購買・運用の複雑さを排除し、セキュリティ担当者の生産性を向上

### 次のステップ

1. **現状セキュリティ態勢の評価**
   - 既存ツール、導入すべきレイヤーを確認

2. **パイロットプロジェクトの実施**
   - 1～2 のパートナーソリューションから試験導入

3. **本番環境への段階的展開**
   - 成功事例を基に、組織全体への展開を計画
