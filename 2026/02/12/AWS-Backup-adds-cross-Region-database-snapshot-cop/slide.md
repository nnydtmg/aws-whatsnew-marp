# AWS Backup
クロスリージョン データベーススナップショット機能

**発表日: 2026年2月12日**

---

## 概要

### クロスリージョン論理エアギャップコンテナへのワンステップコピー

- **対応データベース**: Amazon Aurora、Amazon Neptune、Amazon DocumentDB
- **新機能**: 単一アクション型のクロスリージョン スナップショットコピー
- **効果**: 復旧時間目標（RPO）の短縮、コスト削減、自動化

---

## 前提・背景

### 従来の課題

- **2ステップの複雑なプロセス**
  - ステップ1: ターゲットリージョンのバックアップコンテナへコピー
  - ステップ2: 同じリージョン内の論理的エアギャップコンテナへコピー
  
- **運用負荷**
  - 中間コピー状態を監視するためのカスタムスクリプトが必要
  - AWS Lambda関数などの追加実装が必要

### 事業背景

- ランサムウェア攻撃のリスク増加
- リージョン障害への対応需要の高まり
- 本番環境のセキュリティ強化要求

---

## 変更内容・新機能

### 1. ワンステップコピー機能

- 従来の2ステップ → **1ステップで完了**
- クロスリージョン から 論理エアギャップコンテナへ直接コピー
- AWS Management Console、CLI、SDKで利用可能

### 2. 対応リソース

<div class="columns">
<div>

#### サポート対象

- **Amazon Aurora**
- **Amazon Neptune**
- **Amazon DocumentDB**

</div>
<div>

#### サポート範囲

- AWS Backupがサポートする全リージョン
- すべてのスナップショットタイプ

</div>

## 効果・メリット

### 運用効率化

- ✅ **復旧時間目標（RPO）の短縮** - より迅速な復旧を実現
- ✅ **コスト削減** - 中間コピーに関連するコストを排除
- ✅ **スクリプト不要** - カスタムスクリプト/Lambda関数が不要

### セキュリティ向上

- ✅ **ランサムウェア対策** - エアギャップで隔離されたバックアップ
- ✅ **リージョン障害対策** - 異なるリージョンへの自動保護
- ✅ **簡潔なワークフロー** - ミスの削減

### 1. 金融機関

- RPOの短縮による復旧時間の最小化

### 2. エンタープライズ向けアプリケーション

- 複数リージョンでの本番DB保護
- ランサムウェア対策と事業継続性確保

### 3. SaaS企業

- マルチテナント環境での顧客データ保護

## 利用開始方法

### 前提条件

- AWS BackupでサポートするAurora/Neptune/DocumentDB
- 論理的にエアギャップされたコンテナが別リージョンに存在

### 手順

1. **AWS管理コンソール** にログイン
2. **AWS Backup** → バックアッププランを設定
3. **クロスリージョン コピー先** として論理エアギャップコンテナを選択
4. **保護を開始**

### 利用可能なインターフェース

- AWS Management Console
- AWS CLI（`start-copy-job` コマンド）
- AWS SDKs

---

## 技術詳細

### コピープロセス

```
ソースリージョン (Aurora/Neptune/DocumentDB)
    ↓
[ワンステップコピー処理]
    ↓
ターゲットリージョン の 論理エアギャップコンテナ
```

### セキュリティ特性

- **論理的エアギャップ**: 通常のバックアップからネットワーク的に隔離
- **イミュータブル**: コピー後の改ざん防止
- **暗号化**: トランジット中も保存時も暗号化

## まとめ

### AWS Backupの進化

| 項目 | 従来 | 新機能 |
|-----|-----|-------|
| **クロスリージョン コピー** | 2ステップ | 1ステップ ✨ |
| **必要なカスタムコード** | あり | 不要 ✨ |
| **RPO** | 長い | 短い ✨ |
| **中間コスト** | あり | なし ✨ |

### 推奨される次のステップ

1. 既存バックアップ戦略を評価
2. 論理的にエアギャップコンテナを設定
3. パイロットプロジェクトで検証
4. 本番環境への段階的展開
