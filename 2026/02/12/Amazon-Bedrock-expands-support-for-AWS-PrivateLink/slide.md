# Amazon Bedrock が AWS PrivateLink に対応を拡張
Project Mantle ベースの bedrock-mantle エンドポイント

**発表日: 2026年2月12日**

---

## 概要

### AWS PrivateLink 対応の拡張

- **新エンドポイント**: bedrock-mantle エンドポイントが AWS PrivateLink に対応
- **プライベートアクセス**: VPC 内からセキュアにアクセス可能
- **OpenAI API 互換性**: OpenAI API 形式で統一されたインターフェース
- **グローバルリージョン対応**: 複数のAWSリージョンで利用可能

---

## 前提・背景

### エンタープライズの課題

- 生成AI アプリケーションのセキュリティ要件の増加
- ネットワーク分離とプライベート接続の必要性
- 複数のモデルサービス提供者との統合

### Project Mantle の役割

- 新しい分散推論エンジン
- モデルのオンボーディング簡素化
- 高性能でスケーラブルなサーバーレス推論
- OpenAI API との互換性を提供

---

## 変更内容・新機能

### 1. AWS PrivateLink サポート

- bedrock-mantle エンドポイント向けの VPC エンドポイント作成が可能
- インターネットゲートウェイを経由しない直接接続
- AWS PrivateLink インターフェースエンドポイント経由でアクセス

### 2. Project Mantle エンジン

- 大規模機械学習モデルサービングの実現
- 新しいモデルの高速オンボーディング
- 自動スケーリングと効率的なリソース管理

### 3. OpenAI API 互換性

- 統一されたAPI インターフェース
- デフォルトの顧客クォータを高く設定
- 統合プールによる効率的な容量管理

## メリット・効果

### セキュリティ面

- **プライベートネットワーク**: インターネットを経由しない接続
- **ネットワーク分離**: 企業のセキュリティ要件を満たす
- **エンタープライズグレード**: 高度なアクセス制御が可能

### 運用面

- **統一インターフェース**: OpenAI API 準拠で移行が容易
- **自動容量管理**: デフォルトクォータが高く、スケーリング対応
- **統合プール**: 複数のエンドポイント間でのリソース最適化

### コスト効率

- **サーバーレス**: インフラ運用の削減
- **自動スケーリング**: 需要に応じた柔軟な課金
