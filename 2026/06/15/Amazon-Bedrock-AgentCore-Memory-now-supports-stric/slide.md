# Amazon Bedrock AgentCore Memory が厳密に一貫性のあるメタデータをサポート

Amazon Bedrock AgentCore Memory now supports strictly consistent metadata for long-term memory

**発表日: 2026年6月15日**

---

## 概要

### 3つの主要な改善点

- **直接メタデータ添付**: LLM推論を経ずにアプリケーションから直接メタデータ値を添付可能
- **厳密な一貫性保証**: STRICTLY_CONSISTENT設定で値の正確な転送と保持を実現
- **エンタープライズ対応**: 部門別検索、コンプライアンス、マルチテナント環境での活用

---

## 前提・背景

### これまでの課題

- メタデータ値はLLMによる推論でのみ抽出され、精度や一貫性に課題があった
- 複雑なメタデータ管理が必要なエンタープライズ環境での制限
- コンプライアンスやマルチテナント対応の困難さ

### 市場動向

- AgentCore Memory の長期メモリ機能が段階的に強化（メタデータ、ストリーミング通知など）
- エンタープライズAIの信頼性と予測可能性への需要増加
- 構造化データと非構造化データの融合ニーズ

---

## 主な機能と改善

### 1. 厳密に一貫性のあるメタデータ（STRICTLY_CONSISTENT）

- アプリケーションから直接メタデータ値を提供
- 短期メモリから長期メモリへの値の変更なし転送
- STRING型キー、インデックス付きキー定義が必須

### 2. メタデータのグループ化と統合ルール

- 同じメタデータ値を持つイベントは一緒に抽出・統合
- 異なる値を持つレコードは決してマージされない
- イベント分離による正確なメモリ管理

### 3. 設定の柔軟性

- 戦略ごとに最大3つの厳密に一貫性のあるキーを設定可能
- セマンティック、ユーザー設定、エピソディック戦略をサポート
- カスタムオーバーライド対応

---

## ユースケース

### 部門別スコープ検索

エージェントが異なる部門のデータを独立して検索・処理

### コンプライアンス管理

規制対象レコードと標準レコード間の厳密な境界管理

### マルチテナント環境

各テナントのデータが独立して処理され、テナント間の混在を防止

---

## 効果・メリット

- **データ精度向上**: LLM推論を経ないため、メタデータの正確性が保証される
- **運用効率化**: 複雑なメタデータ管理ロジックの簡素化
- **規制対応**: コンプライアンス要件への確実な対応
- **スケーラビリティ**: マルチテナント環境での確実なデータ分離
- **予測可能性**: メモリ統合ルールが明確で予測可能な動作

---

## 利用可能なリージョン

- Amazon Bedrock AgentCore Memory がサポートされているすべてのAWSリージョン
- 主要リージョン: 米国東部、欧州、アジア太平洋地域ほか
