# AWS CloudFormationとCDKがデプロイ前検証で開発フィードバックループを高速化

CloudFormationとCDKのデプロイ前検証機能

**発表日: 2026年6月30日**

---

## 概要

### AWS CloudFormationの新しいデプロイ前検証機能

- **自動検証実行**: Create StackおよびUpdate Stack操作時に自動的にデプロイ前検証を実行
- **数秒での検出**: リソースプロビジョニング前に一般的なデプロイエラーを数秒で検出
- **開発速度の加速**: プロビジョニングとロールバックのサイクルを待つ必要がなくなる

---

## 前提・背景

### 従来の課題

- デプロイエラーはリソースプロビジョニング後に発生することが多い
- エラー検出までに長時間待機が必要
- プロビジョニングとロールバックのサイクルに時間がかかる

### 市場動向

- DevOps/CI/CDパイプラインの高速化への需要
- インフラストラクチャ as Code の一般化
- AIエージェントによる自動プロビジョニングの登場

---

## 主な機能

### 検証対象の構文エラー

- **プロパティ構文エラー**: テンプレートの構文チェック
- **リソース名の競合**: リソース名の重複検出
- **S3バケットの制約**: S3バケット関連の制約チェック

### 新規追加の3つの検証チェック

- **サービスクォータ制限検証**: リージョンのサービスクォータ超過を事前検出
- **AWS Config Recorder競合検出**: AWS Config Recorderの設定競合をチェック
- **ECRリポジトリ削除準備状況検証**: リポジトリ削除準備状況を確認

---

## CDKでの利用方法

### cdk deployコマンド

- 構造化された検証結果をレポートとして表示
- デプロイ前に検証エラーを確認可能

### cdk validateコマンド

- CloudFormationテンプレートの検証を実行
- スタンドアロンで検証結果を確認可能

---

## エラー確認方法

### 確認方法

- **DescribeEvents API**: プログラマティックにエラーを取得
- **CloudFormationコンソール**: Events タブから検証エラーを表示

### 無効化オプション

- **DisableValidationパラメータ**: CloudFormation APIで検証をスキップ
- **--disable-validationフラグ**: CDK CLIで検証をスキップ
