# AWS Sign-in がリソースベースポリシーとリソース制御ポリシーに対応

ネットワークレベルとアカウントレベルのコンソールアクセス制御を実現

**発表日: 2026年6月16日**

---

## 概要

### 主な変更点

- **リソースベースポリシー対応**: 個別のAWSアカウント単位でコンソールサインインを制限
- **リソース制御ポリシー（RCP）対応**: AWS Organizations経由で組織全体にポリシーを適用
- **ネットワークレベルの制御**: サインイン時に期待されるネットワークのみにアクセスを制限
- **継続的な評価**: セッション中の新しい認証情報リクエスト時にもポリシーを評価

---

## 前提・背景

### 従来の課題

- コンソールアクセス制御が限定的で、ネットワークレベルの制限ができなかった
- 組織内のセキュリティ要件を統一的に適用することが困難
- ユーザーのアクセス可能なアカウントとネットワークを細かく制御できなかった

### 市場の動向

- AWS IAM の進化により、より細粒度の権限管理が求められている
- リソース制御ポリシー（RCP）の活用が組織レベルのセキュリティ強化の鍵に
- ゼロトラスト・アーキテクチャの重要性が高まる中、ネットワークベースの制御が必須

---

## 主な機能

### 1. リソースベースポリシー

- **適用範囲**: 個別のAWSアカウント
- **制御単位**: アカウントレベルでのコンソールサインイン制限
- **ユースケース**: 特定アカウントへのアクセスをネットワークに基づいて制限

### 2. リソース制御ポリシー（RCP）

- **適用範囲**: AWS Organizations経由で組織全体
- **制御単位**: 組織内すべてのアカウントへの統一的なアクセス制御
- **管理**: AWS Organizations管理アカウントから一元管理

### 3. AWS Management Console Private Access との連携

- 既存のプライベートアクセス機能と組み合わせて使用可能
- より層厚いセキュリティ対策を構築

---

## 効果・メリット

### セキュリティの強化

- **多層防御**: ネットワークとアカウント両レベルでのアクセス制御
- **ポリシー実装の簡素化**: 組織全体に統一されたポリシーを一元管理
- **コンプライアンス対応**: 組織の厳格なセキュリティ要件を自動的に実装

### 運用効率の向上

- **追加コストなし**: すべてのAWSコマーシャルリージョンで利用可能
- **継続的な評価**: セッション中のポリシー評価により最新のセキュリティルールを適用
- **既存環境への統合**: AWS Management Console Private Accessと組み合わせた統合的な運用

---

## ユースケース

### 金融機関

- コンプライアンス要件に基づき、特定ネットワークからのみコンソールアクセスを許可
- 組織内のすべてのAWSアカウントに統一されたポリシーを適用

### エンタープライズ組織

- 子会社や事業部のアカウントに対し、親会社のセキュリティポリシーを組織レベルで強制
- リモートワーク環境でのアクセス管理を厳格に制御

### マルチアカウント環境

- 本社ネットワークからのアクセスのみを許可
- 特定のVPN接続やプライベートネットワークを経由したアクセスに限定

---

## まとめ

### AWS Sign-in セキュリティ機能の進化

- **ネットワーク + アカウントレベルの統合制御**: ゼロトラストアーキテクチャへの対応
- **組織全体への統一的なポリシー適用**: AWS Organizationsとの統合強化
- **既存サービスとの統合**: AWS Management Console Private Accessとの連携で多層防御を実現

### 次のステップ

3. テスト環境でのポリシー設定・検証
4. 本番環境への段階的な展開
