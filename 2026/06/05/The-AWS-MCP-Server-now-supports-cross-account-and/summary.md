# AWS MCP Serverがクロスアカウント・クロスロールアクセスに対応

The AWS MCP Server now supports cross-account and cross-role access

**カテゴリ:** What's New
**公開日:** 2026-06-05T07:00:00
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-mcp-server/)

このページでは、AWS What's Newで発表された「The AWS MCP Server now supports cross-account and cross-role access」の内容を日本語で要約し、スライド形式で確認できます。

---

## 要約

AWS MCP Server が複数の AWS アカウントおよび IAM ロール間でのシームレスなクロスアカウント・クロスロールアクセスに対応するようになり、AI コーディングエージェントがセッション再起動なしに複数のアカウント間を効率的に切り替えることが可能になりました。このアップデートは、マルチアカウント環境で複数のサービスを管理する DevOps エンジニアおよびアプリケーション開発者にとって特に有用です。

## このアップデートで何が変わったか

- 従来は、プロファイルの切り替えのたびに AI コーディングセッションを停止し、ローカル AWS 認証情報を更新し、MCP サーバーを再起動する必要がありました。

- 新機能では、各コマンドでプロファイルを指定することにより、ユーザーはアカウントとロール間をシームレスに切り替えることができます。

## 対象ユーザー

## 詳細

- AWS MCP Server が複数の AWS アカウントおよび IAM ロール間でのクロスアカウント・クロスロールアクセスに対応するようになりました。

- 単一のセッション内で、AI コーディングエージェント（Kiro、Claude Code、Codex など）がサーバーの再起動なしに複数のアカウントとロールを切り替えることができるようになりました。

- 従来は、プロファイルの切り替えのたびに AI コーディングセッションを停止し、ローカル AWS 認証情報を更新し、MCP サーバーを再起動する必要がありました。

- 新機能では、各コマンドでプロファイルを指定することにより、ユーザーはアカウントとロール間をシームレスに切り替えることができます。

- 各リクエストが使用するプロファイルを指定するため、コマンドが誤ったアカウントに到達するリスクはありません。

- DevOps エンジニアは、本番環境とステージング環境のアカウント間で CloudWatch ログをクエリして問題を診断したり、アプリケーション開発者は一つのアカウントで Lambda 設定を更新し、別のアカウントで S3 バケットポリシーを調整したりすることができます。

- この機能は US East（N. Virginia）および Europe（Frankfurt）リージョンで利用可能です。

- このアップデートは、マルチアカウント環境で作業する開発者および DevOps エンジニアに特に有益です。
