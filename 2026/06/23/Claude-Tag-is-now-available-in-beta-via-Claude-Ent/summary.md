# Claude Tag が AWS Marketplace 経由で Claude Enterprise のベータ版として利用開始

Claude Tag is now available in beta via Claude Enterprise in AWS Marketplace

**カテゴリ:** What's New
**公開日:** 2026-06-23
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/06/claude-tag-aws-marketplace/)

このページでは、AWS What's Newで発表された「Claude Tag is now available in beta via Claude Enterprise in AWS Marketplace」の内容を日本語で要約しています。

---

## 要約

Claude Tagは、AWS Marketplace経由でClaude Enterpriseにアクセスするお客様向けに、Slackなどのチャネルに直接Claudeを統合できる新しいベータ機能です。チーム全体での協働作業、セキュリティ管理、支出管理を実現する包括的なソリューションとなります。

## このアップデートで何が変わったか

Claude Tagは、Anthropicが新たにベータ版として提供開始した機能であり、AWS Marketplaceを通じてClaude Enterpriseにアクセスするお客様が利用できます。

### 主な変更点

- **Slack への統合**: チームが既に使用しているチャネルに直接 Claude を統合可能
- **@Claude タグ付け**: ユーザーがタスクを委譲し、Claude が自動実行
- **コンテキスト記憶**: チャネル内の関連情報を記憶し、将来のタスク実行を計画
- **アクセス管理**: 選択されたチャネルにアクセス権を付与、ツール・データ・コードベースを接続
- **セキュリティ・ガバナンス**: チャネルごとに独立したアイデンティティで動作
- **支出管理機能**: 消費量ベースで使用状況を追跡、組織全体の予算可視化とチャネルごとの制限

## 対象ユーザー

- AWS Marketplace 経由で Claude Enterprise にアクセスするお客様
- エンタープライズ組織全体（管理者がプロビジョニング可能）
- Slack を活用するチーム・部門

## 利用開始方法

### セットアップの流れ

1. **管理者がプロビジョニング**: Claude 管理コンソールでエージェントアイデンティティを作成（約 1 時間）
2. **チャネルスコープ設定**: 各チャネルのアクセス権と接続ツールを定義
3. **ツール・データ接続**: 必要に応じてツール、データ、コードベースを接続
4. **ユーザー通知**: チーム内で Claude Tag の使用方法を周知

## 主な機能

### 統合機能

- **ネイティブ Slack 統合**: チームが既に使用しているチャネルに直接統合
- **@Claude タグ付け**: ユーザーが @Claude をタグ付けしてタスク委譲
- **コンテキスト保存**: チャネル内の関連情報を記憶・参照
- **アクセス管理**: チャネルごとに細粒度なアクセス制御
- **マルチユーザー対応**: 複数のユーザーが同じ Claude アイデンティティを共有可能

### セキュリティとガバナンス

- **チャネルスコープアイデンティティ**: チャネルごとに独立したアイデンティティで動作
- **支出管理機能**: 消費量ベースで使用状況を追跡
- **予算可視化**: 組織全体の予算管理とチャネルごとの制限設定
- **アンビエントモード**: デフォルトでオフ（セキュリティ重視）

## 期待される効果・メリット

### 生産性向上

- **タスク委譲の効率化**: @Claude でタスクを自動実行、ユーザーは他の業務に集中
- **コンテキスト共有**: チャネル全体で情報・進捗を可視化
- **会話の継続性**: メンバーが途中から会話を引き継ぎ可能
- **多機能連携**: ツール、データ、コードベースを接続して複合的なタスク実行

### 管理・セキュリティ

- **一元管理**: 単一 Claude アイデンティティで全社展開
- **コスト管理**: 使用状況を透明に把握・制御
- **ガバナンス対応**: セキュリティ・支出制限をチャネルごとに設定
