# Amazon Bedrock AgentCore Memory がクロスアカウントアクセスに対応

Amazon Bedrock AgentCore Memory now supports cross-account access

**What's New** | 2026-06-23T17:20:00

---

## 概要

- Amazon Bedrock AgentCore Memoryがクロスアカウントアクセスに対応し、複数のAWSアカウント間でメモリリソースを共有できるようになりました。
- これにより、マルチアカウントアーキテクチャにおいて、リソースベースポリシーを通じたアクセス制御と、別アカウントへの配信先設定が可能になります。

---

## 前提・背景

### マルチアカウント環境の課題

- 複数のAWSアカウントを管理する際、メモリリソースの共有が必要
- エージェントとメモリが異なるアカウントに配置される場合がある
- クロスアカウント間でのセキュアなアクセス制御の必要性

### Amazon Bedrock AgentCore の進化

- Memory Streaming（2026年3月）に続く機能拡張
- セキュリティと運用性の向上
- エンタープライズレベルのマルチテナント対応

---

## 主な機能

### リソースベースポリシーによるアクセス制御

- リソースベースポリシーを使用して、別アカウントのプリンシパルにアクセス権限を付与
- メモリARNを参照することで、別アカウントから安全にアクセス可能

### クロスアカウント配信先の設定

- Amazon S3、Amazon SNS、Amazon Kinesis Data Streamsを別アカウントに配置
- メモリリソースはこれらのサービスにペイロードとストリームイベントを配信可能

### 対応するメモリ操作

- イベント作成
- メモリレコード書き込み
- レコード取得
- セマンティック検索

---

## 利用可能なリージョン

- Amazon Bedrock AgentCore MemoryがサポートされているすべてのAWSリージョンで利用可能

---

## アーキテクチャの利点

### マルチアカウント構成

- メモリリソース専用アカウントとエージェント実行アカウントを分離可能
- リソース管理とコスト配分の最適化
- セキュリティ境界の明確化

### 運用効率の向上

- 複数のワークロード間でメモリリソースを効率的に共有
- 段階的なデプロイメント展開が可能
- ガバナンス要件への対応

## ユースケース

### 複数チームによる共有メモリの利用

- 複数の事業部門が共有のエージェント記憶領域を利用
- 部門ごとのアカウント分離とメモリの一元管理

### エンタープライズ環境での展開

- 金融機関、healthcare など規制対象業務向けの多層アーキテクチャ
- メモリストレージの一元管理と監査ログの統一

### ハイブリッド構成

- 本番環境と開発環境でのメモリ共有
- 段階的なテストと本番リリース
