# AWS Security Agentが脅威モデリング機能をサポート

**発表日: 2026年6月17日**

---

## 概要

### AWS Security Agent に追加された脅威モデリング機能

- **AI駆動型の自動脅威分析**: STRIDEフレームワークを使用して脅威を自動生成
- **開発段階からの統合**: 設計ドキュメントから脅威モデルを作成
- **デプロイ前評価**: セキュリティチームによる包括的な検証
- **パブリックプレビュー**: 追加費用なしで利用可能

---

## 前提・背景

### セキュリティチームの課題

- **時間と専門知識の不足**: 手動の脅威モデリングに多大な労力が必要
- **スケーラビリティの問題**: 複雑なアーキテクチャの脅威分析に対応困難
- **開発ライフサイクル全体への対応**: 設計から本番環境まで一貫した検証が必要

### 市場動向

- **AIエージェント活用の拡大**: 脅威モデリングの自動化が注目される
- **ジェネレーティブAIの活用**: OWASP Top 10 for LLM、STRIDEなどのフレームワークとの組み合わせ
- **セキュアバイデザイン**: 開発の初期段階からセキュリティを組み込む重要性

---

## 主な機能

### 1. 脅威モデリングの自動化

- **ソースコード分析**: アプリケーションの設計ドキュメントとコードを自動分析
- **STRIDE フレームワーク適用**: Spoofing、Tampering、Repudiation、Information Disclosure、Denial of Service、Elevation of Privilege を分類
- **軽減策の提案**: 特定された脅威に対する推奨対策を提供

### 2. 開発ツール統合

- **IDE連携**: Kiro、Claude Code などの開発環境にネイティブ統合
- **早期対応**: 設計段階で脅威に対処可能
- **継続的検証**: プルリクエストレベルでの自動セキュリティレビュー

---

## 対応状況と利用条件

### 提供状況

- **提供形態**: パブリックプレビュー
- **対応リージョン**: AWS Security Agent がサポートするすべてのリージョン
- **価格**: プレビュー期間中は無料

### 活用シーン

- **開発者向け**: IDE統合による設計段階での脅威対応
- **セキュリティチーム向け**: デプロイ前の包括的な脅威評価
- **アーキテクト向け**: アプリケーション設計ドキュメントの自動検証

---

## メリット

### 業務効率化

- 🚀 **開発速度向上**: 脅威分析を週単位から数時間に短縮
- ⚙️ **業務効率化**: 手動作業の自動化によるコスト削減
- 📊 **スケーラビリティ**: 複数プロジェクトの一括管理が可能

### セキュリティ強化

- 🔍 **脅威への早期対応**: 問題が大きくなる前に検出・修正
- 💡 **セキュリティ意識向上**: チーム全体の脅威モデリング理解を促進
- 📈 **包括的な検証**: 設計から本番まで全ライフサイクルをカバー

---

## ユースケース

### 1. マイクロサービスアーキテクチャ

複雑なマイクロサービス環境における各サービス間の脅威を自動分析し、API セキュリティ設定を最適化

### 2. ジェネレーティブAI アプリケーション

LLM統合アプリケーションのプロンプトインジェクション、データ漏洩リスクを設計段階で把握

### 3. 規制対応

金融機関、医療機関の脅威モデリング要件を自動化して満たし、コンプライアンス対応を加速

---

## まとめ

### AWS Security Agent 脅威モデリング機能の価値

- **セキュアバイデザイン**: 開発ライフサイクル全体にセキュリティを統合
- **自動化による効率化**: 脅威分析の時間を大幅に短縮
- **開発・セキュリティチームの協調**: 共通言語での脅威議論を実現

### 次のステップ

1. **評価開始**: パブリックプレビューで自社のアプリケーションで試験
2. **チーム教育**: 脅威モデリングの基礎知識を組織全体で共有
3. **段階的展開**: パイロットプロジェクトから本番運用への移行
