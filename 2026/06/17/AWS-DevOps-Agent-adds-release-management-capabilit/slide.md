# AWS DevOps Agent でリリース管理機能がプレビュー提供開始

AWS DevOps Agent adds release management capability (preview)

**What's New** | 2026-06-17T15:00:00

---

## 概要

- AWS DevOps Agentのリリース管理機能プレビューにより、コード変更の本番環境対応性を自動評価し、自律的なリリーステストを実行することで、安全で確実なコード配備が可能になります。
- このアップデートは、デリバリーとオペレーション全体を通じてチームの配備速度を加速し、運用上の優秀性を実現するのに適しています。

---

## 前提・背景

### これまでの課題

- AWS DevOps Agentがリリース管理機能をプレビューで追加したことが発表されました。
- リリース準備状況のレビュー機能により、コード変更が本番環境の安全性基準に適合しているかを評価します。
- 内部標準からのずれ、依存関係への影響、アクセス制御をチェックする機能が提供されます。
- リポジトリ間の依存関係をマッピングして、コミット前に破壊的な変更を検出します。
- AWS Well-Architected ベストプラクティスに基づいて、インフラストラクチャ変更が適合しているかを確認します。

### 関連する最近の動向

- **AWS DevOps Agent GA リリース**（2026年3月）: より多くのユースケース、統合、エンタープライズ機能を追加
- **インシデント対応の自動化**: AWS DevOps Agentがシステム信頼性向上に貢献
- **開発効率向上**: GitLabなどのツールとの統合による開発時間短縮

---

## 変更内容・新機能

### 1. リリース準備状況のレビュー機能

- コード変更が本番環境の安全性基準に適合しているかを自動評価
- 内部標準からのずれを検出
- 依存関係への影響を分析
- アクセス制御をチェック

### 2. 依存関係マッピング

- リポジトリ間の依存関係を自動マッピング
- コミット前に破壊的な変更を検出
- 連鎖的な影響を事前に把握

### 3. リリーステスト機能

- ウェブおよびAPIベースのアプリケーション向けテストプランを自動生成・実行
- 回帰テストを自動実行
- UXの問題を検出
- 統合テストの失敗を検出
- 人間のレビュアーが見落とす可能性のある問題をキャッチ

---

## 効果・メリット

- **安全で確実なコード配備**: 本番環境対応性を自動評価
- **配備速度の加速**: チームがより迅速にコードをリリース可能
- **MTTRの削減**: 問題検出から修正までの時間短縮
- **運用上の優秀性**: ベストプラクティスに準拠したデプロイメント
- **プレビュー期間は無料**: 追加コストなしで新機能を試用可能
- **段階的な導入**: デリバリーとオペレーションの両方のチームに対応
