# AWS Security Agent が Kiro Power と Claude Code をサポート
IDE統合によるセキュリティスキャンと脅威検証が実現

**発表日: 2026年6月17日**

---

## 概要

- **IDE統合の強化**: Kiro Power と Claude Code プラグインでセキュリティスキャンを IDE 内から直接実行
- **シミュレーション検証**: サンドボックス環境で検出結果を検証し、誤検知を最小化
- **プラットフォーム統合の拡大**: GitLab、GitHub Enterprise、Bitbucket、Confluence に対応
- **開発者ファーストなアプローチ**: IDE から離れることなく脅威モデル、コードスキャン、修復を実行

---

## 前提・背景

### 開発セキュリティの課題

- **開発効率と脅威検証の分離**: 従来はセキュリティツールが IDE 外で動作
- **誤検知による生産性低下**: 検証されていない検出結果に対する対応コスト
- **複雑なツール連携**: 複数のプラットフォームに対応する手間

### 市場の動向

- **AI エージェント IDE の成長**: Kiro (AWS) と Claude Code による開発方式の急速な普及
- **エンタープライズ向けセキュリティ**: 企業は GitHub Enterprise、GitLab Self-Managed など多様なプラットフォームを採用
- **DevSecOps の加速**: セキュリティを開発ライフサイクル全体に組み込む必要性

---

## 主な新機能

### 1. Kiro Power / Claude Code プラグイン

- IDE 内から AWS Security Agent を直接実行
- セキュリティスキャンをプロンプトで制御可能
- MCP (Model Context Protocol) による統合

### 2. シミュレートされた検証

- コードスキャナーの検出結果をサンドボックス環境で再現
- エクスプロイト可能性の証拠を提供
- 誤検知を大幅に削減

## プラットフォーム統合の拡大

### 対応プラットフォーム

| プラットフォーム | バージョン |
|---------------|--------|
| GitHub | Enterprise / SaaS |
| GitLab | Self-Managed / SaaS |
| Bitbucket | Self-Hosted / Cloud |
| Confluence | ドキュメント参照用 |

- 既存のソースコントロールプラットフォームにシームレスに接続
- 脅威モデルの構築から修復までの一連の作業を IDE 内で完結

---

## ユースケース

### 1. セキュリティファーストな開発フロー

- 開発者が IDE で機能を実装
- Kiro Power / Claude Code で自動セキュリティレビュー
- 検証済みの脅威情報に基づいて修復を実施

### 2. セキュリティチームのワークフロー

- 検証済みの検出結果に優先順位を付与
- 誤検知による無駄な対応を削減
- DevOps Agent との連携で本番環境への影響を事前把握

---

## 効果・メリット

### 開発者向け

- **IDE 統合**: セキュリティツールを IDE に統合し、コンテキストスイッチを削減
- **検証済みの脅威情報**: エクスプロイト可能性の証拠を提供し、優先順位付けを容易化
- **多言語対応**: 複数のプラットフォームを統一インターフェースで管理

### セキュリティチーム向け

- **誤検知の削減**: シミュレーション検証により、対応コストを大幅削減
- **リアルタイム可視化**: IDE 内でセキュリティ検証が実行でき、進捗状況を把握
- **エンタープライズ対応**: 多様なプラットフォームに対応し、エンタープライズ環境を支援

---

## 利用可能性

- **リージョン**: AWS Security Agent が対応するすべてのリージョンで利用可能
- **対応リージョン**: 米国東部 (バージニア北部) など主要リージョンで即時利用可能
- **段階的展開**: 追加リージョンへの展開予定

- **開発とセキュリティの融合**: DevSecOps の次段階へ
- **エンタープライズ対応**: 複雑なプラットフォーム環境での統一的なセキュリティ管理
