# AWS IAM Identity Center が AWS アカウントとアプリケーション向けの独立したクォータに対応

AWS IAM Identity Center now supports separate quotas for AWS accounts and applications

**発表日: 2026年6月22日**

---

## 概要

### AWS IAM Identity Center の主な改善

- **独立したクォータシステム**: AWS アカウントとアプリケーション向けに、それぞれ独立した割り当て制限（デフォルトは各7,000個）をサポート
- **容量の有効活用**: 一方のリソースをより多く使用しても、もう一方の容量を消費しない設計
- **スケーラビリティ向上**: 大規模な AWS アカウント環境を管理する組織では、アカウント容量を消費することなくアプリケーションをオンボーディング可能に

---

## 前提・背景

### 従来の課題

- AWS IAM Identity Center を使用する大企業では、AWS アカウントとアプリケーションの管理を同じクォータで行っていた
- 数千のアカウントを保有する組織では、新しいアプリケーションの追加が困難になるケースが発生
- スケーラビリティに関する制限が運用上のボトルネックに

### 業界の動向

- AWS IAM は2026年5月に、ロール、ロール信頼ポリシー、インスタンスプロファイル、マネージドポリシー、アイデンティティプロバイダーなど複数のリソースのクォータを引き上げている
- エンタープライズ環境では ID 管理の複雑化に伴い、より柔軟なクォータシステムが必要とされている

---

## 変更内容・新機能

### 1. 独立したクォータ制限

- **AWS アカウントクォータ**: IAM Identity Center インスタンスあたり最大 7,000 個
- **アプリケーションクォータ**: IAM Identity Center インスタンスあたり最大 7,000 個
- **相互の干渉なし**: 各リソースが独立して機能

### 2. クォータの柔軟な拡張

- AWS Service Quotas コンソールを通じて、必要に応じてクォータ増加リクエストを送信可能
- 既存の高い制限を持つ顧客には、アカウントとアプリケーションの両方に対して同じ制限が自動的に付与される

### 3. 利用可能なリージョン

- このアップデートは、AWS IAM Identity Center が利用可能なすべての AWS リージョンで利用可能

---

## 効果・メリット

- **運用効率の向上**: 複数の リソースのスケーリングを独立して管理可能
- **アプリケーション統合の加速**: アカウント容量の心配なく、必要な数のアプリケーションをオンボーディング
- **大規模環境への対応**: 数千の AWS アカウントを管理する組織にとって、実質的な利便性向上
- **カスタマイズ性**: 自社の要件に応じたクォータ設定が可能

---

## ユースケース

### 大規模エンタープライズ

数千の AWS アカウント環境を保有する組織が、アプリケーション統合フローを加速化できます

### マルチテナント環境

SaaS 提供者は、顧客数が増加してもアカウント管理に影響を受けることなく、アプリケーションスロットを追加可能

### クラウド移行プロジェクト

段階的なアプリケーション移行において、既存のアカウント構成を維持しながら新規アプリケーションを統合

---

## まとめ

### AWS IAM Identity Center のアップデート

- AWS アカウントとアプリケーションのための独立したクォータシステムが実装
- デフォルト 7,000 個ずつのクォータで、スケーラビリティが大幅に向上
- AWS Service Quotas を通じた柔軟な拡張オプションを提供
- すべての AWS リージョンで利用可能

### 次のステップ

1. 自社の AWS アカウント / アプリケーション数を確認
2. 現在のクォータ利用状況を Service Quotas コンソールで確認
3. 必要に応じてクォータ増加リクエストを検討
4. IAM Identity Center への新規アプリケーション統合を加速
