# Amazon Route 53 Global Resolver
インターネット規模のセキュアなDNS解決

**発表日: 2026年3月9日**

---

## 概要

### Amazon Route 53 Global Resolver の3つの主要特徴

- **グローバルなエニーキャストDNS**: 30のAWSリージョンから、どこからでもセキュアなDNS解決を提供
- **高度なDNS脅威フィルタリング**: マルウェア、DGA、DNSトンネリングなどの脅威から保護
- **集中化されたクエリログ**: 組織全体のDNS活動を一元的に監視・監査

### 主な対応範囲

- **IPv4/IPv6**: 両プロトコルのDNSクエリトラフィックに対応
- **30のAWSリージョン**: グローバルなカバレッジで高可用性を実現
- **新規顧客向け**: 30日間の無料トライアル

---

## 前提・背景

### 組織のDNS管理の課題

- オンプレミス、ブランチオフィス、リモートクライアントからのDNS管理が分散化
- DNSセキュリティポリシーの統一が困難
- DNS関連の脅威（データ流出、マルウェア通信）への対応強化が必要
- 複数のDNS解決ソリューション間での運用コスト

### 市場動向

- エンタープライズのハイブリッド・マルチロケーション化の加速
- DNSを悪用した高度なサイバー攻撃の増加
- ゼロトラスト・セキュリティモデルの採用拡大

---

## 主な機能と改善

### 1. グローバルなエニーキャストDNS解決

- **どこからでもアクセス可能**: オンプレミス、ブランチ、リモートロケーション
- **パブリック/プライベート対応**: インターネットドメイン + Route 53プライベートホストゾーン
- **高可用性**: 複数リージョンでのアクティブ・アクティブ構成

### 2. 高度なDNS脅威フィルタリング

- **DGA（Domain Generation Algorithm）検出**: マルウェアのコマンド&コントロール通信をブロック
- **DNSトンネリング検出**: DNS経由のデータ流出、不正通信を遮断
- **AWS Managed Domain Lists**: マルウェア、スパム、フィッシング、成人向けコンテンツなど複数カテゴリーで分類
- **Dictionary DGA保護**: 一般提供に伴い新たに追加

## 主な機能と改善（続き）

### 3. 統合化されたセキュリティ管理

- **DNS Firewall統合**: Route 53 Resolver DNS Firewallと同等の機能
- **暗号化DNS対応**: DNS-over-HTTPS (DoH) / DNS-over-TLS (DoT) サポート
- **DNSSEC検証**: パブリックDNS応答の真正性確認

### 4. 集中化されたクエリログと監視

- **一元的なログ管理**: すべてのクライアントのDNSクエリを中央に集約
- **詳細なレポート生成**: セキュリティ・コンプライアンス監査対応
- **ポリシー遵守の可視化**: 組織全体のDNSアクティビティの把握

## ユースケース

### 1. グローバルエンタープライズの統一DNS管理

**シナリオ**: 複数国のオフィス + クラウドインフラを持つ企業

- 各ロケーションからのDNSクエリを一元管理
- グローバルなDNS脅威フィルタリングポリシーを統一
- コンプライアンス要件に基づいたアクセス制御

### 2. ハイブリッドクラウド環境のセキュアなDNS

**シナリオ**: オンプレミス + AWS + マルチクラウド環境

- AWS Route 53プライベートホストゾーン と オンプレミスDNS の統合解決
- 暗号化DNS (DoH/DoT) による安全な通信
