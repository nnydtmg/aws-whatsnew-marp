# AWS AppConfig adds enhanced targeting during feature flag rollout

**カテゴリ:** What's New
**公開日:** 2026-03-26
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/03/appconfig-enhanced-targeting-feature-flag-rollout/](https://aws.amazon.com/about-aws/whats-new/2026/03/appconfig-enhanced-targeting-feature-flag-rollout/)

---

## 要約

AWS AppConfigは、段階的ロールアウト中に特定のユーザーセグメントに対してフィーチャーフラグとコンフィグレーションデータを細粒度でターゲティングできる強化されたターゲティング機能を提供するようになりました。この機能により、顧客はより安全で制御されたデプロイメントを実現することができます。

## 詳細

- AWS AppConfigが機能フラグとコンフィグレーションデータの段階的ロールアウト中に、特定のセグメントまたは個別ユーザーをターゲットにする強化されたターゲティング機能を追加しました。
- この新機能により、顧客は数分から数時間かけて段階的にフィーチャーフラグやコンフィグレーションデータの変更をロールアウトすることができます。
- AppConfig Agentを使用して、個別ユーザーIDまたは複数のユーザーIDを含む細粒度の制御が可能になり、ロールアウト中に特定のターゲットセグメントに対して機能フラグが「固定」されます。
- この機能により、予期しない変更の影響を制限し、より安全なデプロイメントが実現されます。
- この新しいターゲティング機能は、AWS GovCloud（US）リージョンを含むすべてのAWSリージョンで利用可能です。
