# Amazon Bedrock AgentCore Runtime
管理されたセッションストレージで永続的なエージェント状態を実現

**発表日: 2026年3月25日**

---

## 概要

### 3つの主要改善

- **自動状態永続化**: エージェントのセッション停止後もファイルシステムの状態が自動的に保持
- **開発の簡素化**: チェックポイントロジックやセーブ・リストアコード実装が不要
- **シームレスな継続**: セッションID指定でコード、パッケージ、ビルドアーティファクトがすべて復元

### 公開状況

- **ステータス**: パブリックプレビュー
- **利用可能リージョン**: 14のAWSリージョン（米国、カナダ、アジア太平洋、ヨーロッパ）

---

## 前提・背景

### 従来の課題

- AIエージェントはコード作成、パッケージインストール、アーティファクト生成を行う
- セッション停止時にこれらのファイルシステム上のデータがすべて失われていた
- エージェント開発者が手動で永続化ロジック（チェックポイント、セーブ・リストア）を実装する必要があった
- マイクロVMの一時的な性質により、状態管理が複雑化

### 市場の動向

- AIエージェント技術の急速な普及
- 長期実行エージェントの需要増加
- 開発効率と信頼性の両立が求められている
- Stateful Runtime Environment（AWS + OpenAI共開発）との連携強化

---

## 変更内容・新機能

### 1. マネージドセッションストレージ

- エージェントが指定されたマウントパスに書き込んだすべてのデータが自動永続化
- AgentCore Runtimeが透過的にデータを耐久性ストレージにレプリケート
- セッション停止時にグレースフルシャットダウン中にデータをフラッシュ

### 2. シームレスなセッション再開

- 同じセッションIDで再開時に、新しいマイクロVMが同じストレージをマウント
- ソースファイル、インストール済みパッケージ、ビルドアーティファクト、git履歴がすべて保持
- エージェントは中断した箇所から即座に継続可能

---

## 技術仕様と制限

### ストレージ仕様

| 項目 | 仕様 |  
|------|------|  
| 最大容量 | セッションあたり1GB |  
| ファイルシステム操作 | 標準Linuxファイルシステム（ファイル、ディレクトリ、シンボリックリンク） |  
| データ保持期間 | アイドル時間14日間 |  
| アクセス制御 | 単一セッションデータのみアクセス可能 |  

### セキュリティ

- **隔離**: ストレージ通信は単一セッションに限定
- **分離**: 他のセッションやAgentCore Runtime環境にアクセス不可
- **安全性**: 各セッションが独立した永続ディレクトリを取得

## ユースケース

### 1. 複雑なデータ分析エージェント

- 複数セッションにわたる分析結果とモデルの保持
- インストール済み分析ライブラリの再利用
- 中間結果ファイルの永続化

### 2. ソフトウェア開発支援エージェント

- ソースコードの保持と継続的な改善
- ビルドアーティファクトのキャッシング
- git履歴の保持による変更追跡

### 3. 音声エージェント

- 会話履歴とコンテキスト情報の保持
- 長期にわたるユーザー対話の継続
- セッション間でのユーザープロファイル維持

### 4. エンタープライズAIエージェント

- 機密プロジェクト情報の安全な永続化
- セッション間でのワークフロー状態維持
- 監査ログとアーティファクトの保全
