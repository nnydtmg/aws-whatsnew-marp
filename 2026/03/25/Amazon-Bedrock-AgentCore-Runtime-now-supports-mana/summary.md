# Amazon Bedrock AgentCore Runtime now supports managed session storage for persistent agent filesystem state

**カテゴリ:** What's New  
**公開日:** 2026-03-25  
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/03/bedrock-agentcore-runtime-session-storage/](https://aws.amazon.com/about-aws/whats-new/2026/03/bedrock-agentcore-runtime-session-storage/)

---

## 要約

Amazon Bedrock AgentCore Runtimeが管理されたセッションストレージ機能をパブリックプレビューで提供開始し、エージェントがセッション停止後もファイルシステムの状態を永続化できるようになりました。この機能により、チェックポイントロジックやセーブ・リストアコードの実装が不要となり、エージェント開発がより簡潔になります。

---

## 詳細

### 基本機能

- Amazon Bedrock AgentCore Runtimeが、管理されたセッションストレージ機能をパブリックプレビューで提供開始したことが発表されました
- この機能により、エージェントはストップおよびレジューム サイクル全体を通じて、ファイルシステムの状態を永続化できるようになりました
- エージェントが書き込んだコード、インストールされたパッケージ、生成されたアーティファクト、およびファイルシステムを通じて管理される状態が、セッション停止後も失われなくなります

### 技術的な実装

- セッションストレージを設定すると、各セッションは指定されたマウントパスに永続ディレクトリを取得します
- AgentCore Runtimeが自動的にデータを耐久性のあるストレージにレプリケートします
- セッション再開時に同じセッションIDを使用すると、ソースファイル、インストール済みパッケージ、ビルドアーティファクト、およびgitの履歴がすべて保持された状態で、エージェントは中断したところから継続できます

### 開発効率

- チェックポイントロジックやセーブ・リストア コードの実装が不要です
- エージェント アプリケーションへの変更は必要ありません
- 開発者は状態管理の複雑性から解放されます

### ストレージ仕様

- セッションストレージは、標準的なLinuxファイルシステム操作（通常ファイル、ディレクトリ、シンボリックリンク）をサポートします
- セッションあたり最大1GBのデータを保持できます
- アイドル時間14日間のデータ保持が可能です
- ストレージ通信は単一セッションのデータに限定され、他のセッションやAgentCore Runtime環境にアクセスすることはできません（セキュリティ）

### 利用可能リージョン

この機能は、以下の14のAWSリージョンでパブリックプレビューとして利用可能です：
- **米国**: バージニア北部、オハイオ、オレゴン
- **カナダ**: 中部
- **アジア太平洋**: ムンバイ、ソウル、シンガポール、シドニー、東京
- **ヨーロッパ**: フランクフルト、アイルランド、ロンドン、パリ、ストックホルム

## ユースケース

### 複雑なデータ分析

エージェントが複数セッションにわたって分析結果やモデルを保持し、インストール済みライブラリを再利用できます。

### ソフトウェア開発支援

ソースコード、ビルドアーティファクト、git履歴を保持することで、長期的なコード改善が可能になります。

### 音声エージェント

会話履歴とコンテキスト情報を保持し、ユーザープロファイルを複数セッション間で維持できます。

### エンタープライズAI

機密プロジェクト情報を安全に永続化し、セッション間でワークフロー状態を維持できます。
