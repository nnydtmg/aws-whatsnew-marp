# AWS IAM ロール作成の簡素化
サービスワークフロー内での直接設定が可能に

**発表日: 2026年3月4日**

---

## 概要

### AWS IAM がサービスワークフロー統合に対応

- **ロール作成の一元化**: IAM コンソール移動が不要に
- **シームレスなワークフロー**: ブラウザタブ切り替え廃止
- **権限設定の簡素化**: 新しいパネルで直感的に設定
- **複数サービス対応**: EC2、Lambda、EKS など11サービス対応

---

## 前提・背景

### IAM ロール設定の従来の課題

- IAM コンソールへの頻繁な移動が必要
- ワークフロー中断による効率低下
- ユーザーエクスペリエンスの複雑性
- 開発速度の低下

### 市場のニーズ

- クラウドネイティブ開発の加速
- デベロッパー向けツール体験の向上
- セキュリティと利便性の両立
- ワークフロー自動化への需要増大

---

## 主な機能・変更内容

### 1. インライン IAM ロール作成

- サービスコンソール内で直接ロール作成可能
- 新しいパネルが自動表示
- タブ切り替え不要

### 2. 権限のカスタマイズ方法

| オプション | 特徴 | 用途 |
|-----------|------|------|
| デフォルトポリシー | 推奨権限セット | 標準的なユースケース |
| ステートメントビルダー | 細粒度制御 | 高度なカスタマイズ |

## 効果・メリット

### 開発効率の向上

- **時間短縮**: ワークフロー中断なし
- **ユーザー体験向上**: 直感的なインターフェース
- **エラー削減**: 正しい権限設定を促進

### セキュリティと管理性

- **一時認証情報**: ハードコードキー不要

- 運用工数削減

## 利用シーン（ユースケース）

### 1. EC2 インスタンス立ち上げ時

EC2 管理コンソール → ロール作成ウィザード → 権限設定 → インスタンス起動

### 2. Lambda 関数デプロイ時

Lambda コンソール → ロール作成パネル → カスタム権限設定 → 関数デプロイ

### 3. 複合ワークフロー構築

CloudFormation スタック作成時に必要なロールをその場で作成
