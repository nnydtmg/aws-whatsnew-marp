# Amazon Bedrock AgentCore Memory announces streaming notifications for long-term memory

**カテゴリ:** What's New
**公開日:** 2026-03-12T16:00:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/03/agentcore-memory-streaming-ltm/](https://aws.amazon.com/about-aws/whats-new/2026/03/agentcore-memory-streaming-ltm/)

---

## 要約

Amazon Bedrock AgentCore Memoryがストリーミング通知機能を新たにサポートすることで、メモリレコードの変更をリアルタイムで検出でき、ポーリングロジックの実装が不要になりました。この機能により、開発者はダウンストリームワークフローの自動トリガーやアプリケーション状態の更新、メモリ監査を効率的に実現できるようになります。

## 詳細

### 主な機能

- **ストリーミング通知**: Amazon Bedrock AgentCore Memoryが長期メモリのストリーミング通知機能をサポート
- **プッシュ型通知**: ポーリングの必要性が排除され、メモリレコードが作成または変更されるたびにプッシュ通知を送信
- **Kinesis統合**: 更新内容はAmazon Kinesisに直接ストリーミング

### 効果

- **開発効率向上**: ポーリングロジックやリフレッシュ間隔の管理が不要に
- **ワークフロー自動化**: ダウンストリームワークフローの自動トリガー
- **アプリケーション状態管理**: アプリケーション状態の自動更新
- **監査機能**: メモリ更新を自動的に監査

### コンテキスト

- **長期メモリの役割**: エージェント相互作用から洞察を抽出し、将来の相互作用でパーソナライズされた体験を提供
- **AgentCore Memory**: 完全マネージド型のサービスで、AIエージェントに過去の相互作用を記憶する能力を提供
- **ステートレス解決**: AIエージェントの根本的な課題であるステートレス性を解決

### リージョンサポート

この機能は15のAWSリージョンで利用可能：

- **北米**: 米国東部（バージニア北部）、米国東部（オハイオ州）、米国西部（オレゴン州）、カナダ（中部）
- **ヨーロッパ**: フランクフルト、アイルランド、ロンドン、パリ、ストックホルム
- **アジアパシフィック**: ムンバイ、シンガポール、シドニー、東京、ソウル
- **その他**: 南米（サンパウロ）

### ユースケース例

1. **カスタマーサービス**: 顧客情報の自動更新、過去の相互作用履歴の活用
2. **金融分析**: トランザクション情報の即座の反映、リスク管理の自動トリガー
3. **エンタープライズシステム**: CRMとのシームレス連携、複数システム間のデータ同期

### メリット

- **リソース削減**: ポーリング処理廃止による計算リソース削減
- **応答性向上**: リアルタイムなメモリ変更検出
- **スケーラビリティ**: 大規模ユーザー数への容易な対応
- **運用効率**: 実装と維持管理の複雑さ削減
