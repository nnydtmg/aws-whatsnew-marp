# Amazon EventBridge Scheduler
CreateSchedule API デフォルトクォータ引き上げ

**発表日: 2026年3月4日**

---

## 概要

### 主な変更内容

- **CreateSchedule API のデフォルトクォータが毎秒 5,000 リクエストに引き上げ**
- 11 つの主要 AWS リージョンで利用可能
- 追加のクォータリクエストなしで大規模運用が可能
- 自動スケーリングにより、さらに数万リクエスト/秒まで対応

---

## 前提・背景

### 市場の課題

- **高スループット環境での制約**
  - 大規模スケジュール作成ワークロード向けの上限が不足
  - 追加クォータリクエストの手続きが必要な煩雑さ

- **スケーラビリティ要求の高まり**
  - IoT、ビッグデータ処理など大規模イベント駆動システムの拡大
  - 自動スケーリング環境での適応的なリソース管理が必須

### EventBridge Scheduler の位置づけ

- サーバーレスタスクスケジューラー
- 270以上の AWS サービスと統合可能
- インフラストラクチャ管理が不要
- 数十億単位のスケジュール済みイベントを管理

---

## 変更内容・新機能

### クォータの詳細

| 項目 | 新値 |
|-----|-----|
| CreateSchedule API デフォルトクォータ | **毎秒 5,000 リクエスト** |
| 最大拡張可能 | **数万リクエスト/秒** |

### 対応リージョン（11リージョン）

- **北米**: 米国東部（N. バージニア）、米国東部（オハイオ）、米国西部（オレゴン）
- **欧州**: ヨーロッパ（アイルランド）、ヨーロッパ（フランクフルト）、ヨーロッパ（ロンドン）
- **その他**: 南米（サンパウロ）、アジアパシフィック（ムンバイ）、アジアパシフィック（東京）、アジアパシフィック（シンガポール）、アジアパシフィック（シドニー）

### 機能面での特徴

- **スケジュール作成方式**
  - ワンタイム実行スケジュール
  - 反復実行スケジュール（cron式、レート式、時刻指定）

- **拡張機能**
  - タイムゾーン対応
  - サマータイム自動対応

---

## 効果・メリット

### 運用効率の向上

- ✅ **手続き削減**: クォータリクエスト不要で即座に大規模運用可能
- ✅ **コスト削減**: インフラストラクチャプロビジョニング不要
- ✅ **管理負荷軽減**: サーバーレスで自動スケーリング対応

### スケーラビリティの強化

- ✅ **高スループット対応**: 毎秒 5,000 リクエストから開始
- ✅ **柔軟な拡張**: Service Quotas コンソールで追加拡張が容易
- ✅ **自動適応**: 新しいクォータへのスケーリングは自動実行

### ビジネス上の利点

- ✅ **グローバル展開**: 11 リージョンでの一貫した高パフォーマンス
- ✅ **信頼性向上**: AWS インフラによる高可用性
- ✅ **統合性向上**: 270以上のサービス連携による柔軟なワークフロー

## まとめ

### このアップデートの意義

**Amazon EventBridge Scheduler のスケーラビリティが大幅に向上**
- デフォルトクォータが毎秒 5,000 リクエストに引き上げ
- 11 の主要リージョンで即座に適用
- エンタープライズグレードの高スループット対応が可能に

1. **既存システムの評価**: 現在のスケジュール負荷を確認
2. **パイロット検証**: 新クォータでの性能テスト実施
3. **本番移行計画**: 段階的なワークロード移行の検討
4. **拡張戦略**: 将来の成長に対応する拡張計画の策定
