# AWS MCP Server (Preview)
CloudWatch監視とセマンティック検索機能の追加

**発表日: 2026年3月19日**

---

## 概要

### 3つの主要な改善

- **CloudWatch統合**: 運用メトリクスが自動公開され、リアルタイム監視が可能に
- **セマンティック検索**: Agent SOPsの検出機能が大幅にスケーラブルになった
- **詳細な追跡**: 個別ツールのパフォーマンスメトリクスを細粒度で監視

---

## 前提・背景

### AIエージェント導入の課題

- AIエージェントが複数の AWS タスクを実行する際の可視性不足
- エージェントのパフォーマンス追跡と問題特定の困難さ
- セキュリティと権限の問題を事前に検出することの難しさ

### AWS MCP Server の進化

- プレビュー版でエージェント向けAgent SOPsを順次拡張
- エンタープライズでの運用監視機能の需要が高まり
- CloudWatchとの統合により、既存の監視体系との連携が実現

---

## 主な新機能

### 1. CloudWatch メトリクス自動公開

- **AWS-MCPネームスペース**: CloudWatch上で一元管理
- **追加コストなし**: 既存のCloudWatch環境に統合
- **自動公開**: 設定不要で即座にメトリクスが利用可能

### 2. 監視可能なメトリクス

| メトリクス | 説明 |
|-----------|------|
| **呼び出し数** | エージェントによるツール実行回数 |
| **成功率** | タスク成功の割合 |
| **クライアントエラー** | リクエスト形式の問題 |
| **サーバーエラー** | AWS API側のエラー |
| **スロットリング** | レート制限による制御 |

## 主な新機能（続き）

### 3. ツール単位でのメトリクス追跡

- **call_aws**: AWS API呼び出しの詳細なメトリクス
  - API呼び出し成功率
  - エラーカテゴリ別の集計
  - レスポンスタイム分析

- **retrieve_agent_sop**: Agent SOP検索の実行状況
  - 検索クエリの種類
  - マッチ率と精度
  - 実行時間の分布

### 4. セマンティック検索の強化

- **search_documentation ツール**:
  - 従来: キーワード検索のみ
  - **新**: セマンティック類似性による検索
  - AWS ドキュメント + Agent SOPs を同時検索
  - より関連性の高い結果を自動抽出

---

## 効果・メリット

### 運用監視の強化

- **エージェントの使用パターン追跡**: どのタスクがよく実行されるか可視化
- **権限問題の事前検出**: IAM権限不足を早期に特定
- **エラー率ベースのアラーム設定**: カスタムしきい値で自動通知

### セキュリティと信頼性向上

- **包括的な監査ログ**: すべてのエージェント操作を記録
- **異常な活動の検出**: メトリクスの異常値から問題を検知
- **パフォーマンスの最適化**: ボトルネック箇所の特定と改善

## ユースケース

### 1. インフラ運用チーム

- CloudWatchダッシュボードでエージェント操作を監視
- IAMロール権限の過剰付与を検出
- エージェント導入時の効果測定

### 2. セキュリティチーム

- 不正なAPI呼び出しパターンの検出
- 権限エスカレーション試行の追跡
- コンプライアンス監査ログの生成

### 3. 開発チーム

- Agent SOPs実装時のデバッグ
- エージェント回応性能の測定
- 最適なプロンプト設計の検証

## 利用開始方法

### 前提条件

- AWS アカウント（プレビュー参加者）
- AWS MCP Server プレビュー版のアクセス権
- CloudWatchへのアクセス権限

### 設定手順

1. **CloudWatchを開く**: AWS-MCPネームスペースを確認
2. **ダッシュボード作成**: デフォルトメトリクスで初期設定
3. **アラーム設定**: 重要なメトリクスにしきい値を設定
4. **エージェント実行**: 自動的にメトリクスが記録開始
