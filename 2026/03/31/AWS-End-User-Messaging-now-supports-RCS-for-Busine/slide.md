# AWS End User Messaging
RCS for Business がサポート開始

**発表日: 2026年3月31日**

---

## 概要

### RCS for Business のサポート開始

- **AWS End User Messaging** が **リッチコミュニケーションサービス（RCS）for Business** をサポート開始
- 認証済みのビジネスアイデンティティ（企業名とロゴ）を表示してメッセージ配信
- テキストメッセージングを通知チャネルから **AI駆動型の双方向会話体験** へ進化
- 従来のSMSの課題を解決し、顧客体験を大幅に向上

---

## 前提・背景

### 従来のSMS の限界

- **テキストのみ**: 画像や動的コンテンツを送信できない
- **ビジネス識別なし**: 送信者情報が限定的で詐欺メッセージと区別しづらい
- **一方向通信**: 顧客との双方向対話が困難
- **スケーラビリティ**: 複雑な会話ロジックに対応しきれない

### RCS への期待

- より豊かなメッセージング体験への市場需要の高まり
- ビジネス認証による信頼性の向上
- AI/LLM連携によるカスタマーエンゲージメントの強化
- グローバルなメッセージングスタンダードの統一

---

## 主な機能と改善

### 1. 認証済みビジネスアイデンティティ

- 企業名とロゴをメッセージで表示
- 詐欺・スパムメッセージとの明確な区別
- ユーザー信頼度の向上

### 2. リッチな会話体験

- テキスト、画像、動画、カルーセルのサポート
- インタラクティブなボタンやメニュー
- ワンタイムパスコード、予約リマインダー、配送通知などに対応

### 3. SMSフォールバック機能

- RCS 非対応デバイスでは自動的に **SMS に切り替わり**
- 追加ロジック実装不要で互換性を確保

---

## 技術的な特徴

### RCS エージェント管理

- **AWS End User Messaging コンソール** または **API** で直接作成・管理
- エージェントライフサイクル全体をAWS内で統一管理

### AWS サービス連携

- **Amazon EventBridge**: イベントルーティング
- **Amazon CloudWatch**: ログとメトリクス監視
- **Amazon Data Firehose**: データストレーミング
- **Amazon SNS**: インバウンドメッセージ配信
- **AWS Lambda**: ワークフロー自動化
- **Amazon Bedrock**: AI/LLM エージェント統合

---

## 主要なメリット

### ビジネス面

- **顧客エンゲージメント向上**: リッチコンテンツで訴求力UP
- **信頼性強化**: 認証ビジネスアイデンティティで安心感
- **一貫性**: SMS との自動フォールバックで配信確実性

### 技術面

- **スケーラビリティ**: AWS インフラで数百万メッセージに対応
- **AI統合**: Bedrock 経由で生成AI機能を活用
- **可観測性**: CloudWatch/EventBridge で詳細監視

### コスト効率

- **従量課金制**: 使用量に応じた最適な費用
- **統合管理**: 複数のメッセージングチャネルを一元化

## ユースケース

### 金融サービス

- 認証コード、取引アラート
- パーソナライズされた口座情報提示

### Eコマース・ロジスティクス

- リッチなマーケティングメッセージ
- 配送状況の動的アップデート
- インタラクティブな注文確認

### カスタマーサポート

- AI エージェントによる自動応答
- チャットボット + LLM による高度なサポート

---

## 展開範囲

### 提供リージョン

- **AWS End User Messaging が利用可能なすべてのリージョンで提供**
- グローバル対応で、どの AWS リージョンからでも即座に利用開始可能

---

## まとめ

### AWS End User Messaging の進化

- RCS for Business サポートにより、メッセージングの **質的進化** を実現
- 認証ビジネスアイデンティティ + AI駆動型会話で **顧客体験を革新**
- AWS 統合により **スケーラブルで信頼性の高い** 運用を実現

### 次のステップ

1. 自社のカスタマーエンゲージメント戦略で RCS の活用箇所を検討
2. パイロットプログラムで効果測定
3. AI/Bedrock 連携による対話型体験の試行
4. 本番環境への段階的展開
