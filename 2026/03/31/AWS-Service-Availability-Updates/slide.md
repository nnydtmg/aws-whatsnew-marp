# AWS Service Availability Updates
サービスの可用性に関する重要な更新

**発表日: 2026年3月31日**

---

## 概要

### 3つの変更カテゴリ

- **メンテナンス移行**: 新規顧客向けにアクセス不可（2026年4月30日以降）
- **サンセット対象**: 段階的な廃止予定サービス
- **サポート終了**: 2026年3月31日現在で利用不可

### 重要なポイント

- 既存ユーザーは継続利用可能
- AWS が移行ガイドとサポートを提供
- 早期の移行計画が推奨される

---

<!-- _class: section -->

# 1. メンテナンス移行サービス

段階的なサービス変更

---

## メンテナンス移行サービス

### 対象となるサービス・機能（全9つ）

- **Amazon Application Recovery Controller**
  - Readiness Check 機能が対象

- **Amazon Comprehend**
  - 複数の機能がメンテナンス対象

- **Amazon Rekognition**
  - ストリーミングイベント機能

### 実装スケジュール

| 項目 | 日程 |
|------|------|
| 新規顧客アクセス停止 | 2026年4月30日 |
| 既存顧客 | 継続利用可能 |

<!-- _class: section -->

# 2. サンセット対象サービス

廃止予定のサービス

---

## サンセット対象サービス

### 段階的廃止予定の4つのサービス

- **Amazon RDS Custom for Oracle**
  - エンタープライズデータベース機能

- **Amazon WorkMail**
  - ビジネスメール・カレンダーサービス

- **Amazon WorkSpaces Thin Client**
  - リモートワークステーション管理

- **AWS Service Management Connector**
  - IT サービス管理統合

### 推奨アクション

- 12～18ヶ月の移行期間を想定
- AWS の移行ガイドを参照
- 代替ソリューションへの計画立案

<!-- _class: section -->

# 3. サポート終了機能

---

## サポート終了機能

### Amazon Chime SDK - Proxy Sessions

- **ステータス**: 2026年3月31日時点でサポート終了
- **現在**: 利用不可
- **対象ユーザー**: Proxy Sessions 機能を使用中のお客様

### 対応方法

- AWS ドキュメントで代替ソリューションを確認
- サポートチームに相談

# 5. 移行ガイド・リソース

---

## 移行ガイド・リソース

### AWS が提供するサポート

- **移行ガイド**: サービスごとの詳細ガイド
- **ドキュメント**: 代替ソリューションの情報
- **サポートチーム**: 個別の相談対応
- **ライフサイクルページ**: https://docs.aws.amazon.com/general/latest/gr/service-lifecycle.html

### 推奨ステップ

1. **影響範囲の把握**
   - 使用中のサービスを確認

2. **代替方法の検討**
   - 移行ガイドで代替ソリューションを確認

3. **移行計画の策定**
   - スケジュール・リソースを決定

4. **段階的な移行実施**
   - テスト → 本番切り替え

# 6. まとめ

---

## まとめ

### AWS Service Availability Updates の要点

- AWS は定期的にサービス可用性を見直し
- 3つの変更区分で段階的に対応
- 既存ユーザーへの影響を最小化

### 必要なアクション

- 自社が使用するサービスを確認
- 影響を受ける場合は移行計画を早期に開始
- AWS のサポートとドキュメントを活用

### AWS による支援

- 移行ガイドの提供
- サポートチームの対応
- ドキュメントの充実
