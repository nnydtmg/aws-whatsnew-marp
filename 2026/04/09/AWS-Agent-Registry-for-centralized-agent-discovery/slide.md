# AWS Agent Registry
集中型エージェント検出とガバナンス

**発表日: 2026年4月9日**

---

## 概要

### AWS Agent Registry とは

- **Amazon Bedrock AgentCoreのプレビュー機能**として利用可能
- 組織内のエージェント、ツール、スキルを一元管理する**プライベートなカタログ**
- チームが既存のAI機能を効率的に**検出・再利用**できるプラットフォーム
- 複数のアクセス方法と強力なガバナンス機能を実装
- 包括的な監査証跡を提供

### 主な利点

- AIランドスケープの**完全な可視性**を獲得
- 既に存在する機能の**再構築を回避**
- エンタープライズレベルの**統制と管理**を実現

---

## 前提・背景

### 組織が直面する課題

- **複数のAIエージェント管理の複雑性**: 組織全体で構築されたエージェント、ツール、スキルが散在
- **重複開発の発生**: 既に存在する機能の再構築に時間と資源を浪費
- **検出性の欠如**: 利用可能なAIリソースがチーム間で認識されていない
- **ガバナンスの不備**: アクセス制御と監査のための統一的な仕組みがない

### Amazon Bedrock AgentCore の進化

- 2026年3月: **Policy** が一般提供開始（ガバナンス機能強化）
- 2026年3月: **AgentCore Evaluations** が一般提供開始（品質評価の自動化）
- 2026年3月: **AG-UI プロトコル**対応（ユーザーインタラクション強化）
- 2026年3月: **Stateful MCP** サポート（Model Context Protocol の拡張）
- 2026年4月: **Agent Registry** プレビュー開始（中央集約的な検出と管理）

---

## 主な機能

### 1. リソースの集中管理と検出

- **対応リソース**: エージェント、ツール、スキル、MCPサーバー、カスタムリソース
- **検出方法**: セマンティック検索とキーワード検索の両対応
- **自然言語対応**: 開発者がユースケースを自然言語で説明してエージェントを検出
- **自動検出機能**: URLベースの自動検出でリソース登録を効率化

### 2. アクセス方法の多様性

| アクセス方法 | 説明 |
|-----------|------|
| **AgentCore Console UI** | ウェブベースのUIで直感的に検出・管理 |
| **API (AWS CLI/SDK)** | プログラマティックなアクセス |
| **MCP サーバー** | IDEから直接クエリ可能 |

### 3. セキュリティとアクセス制御

- **IAM ベースのアクセス制御**: AWSの標準的なアイデンティティ管理
- **OAuth（カスタムJWT）対応**: 外部認証システムとの統合
- **きめ細かい権限設定**: リソースごとのアクセス制御が可能

---

## ガバナンスと監査

### 管理機能

- **承認ワークフロー**: 管理者による承認プロセスを実装
- **ガバナンスポリシー適用**: 組織全体のポリシー遵守を強制
- **リソース登録方法**:
  - **手動登録**: コンソール/API経由での明示的な登録
  - **自動検出**: URLベースの自動検出による効率化

### 監査とコンプライアンス

- **AWS CloudTrail 統合**: レジストリアクセスと管理アクション全体をログ記録
- **完全な監査証跡**: 誰が何にアクセスしたかを追跡
- **コンプライアンス対応**: 規制要件への対応を支援
