# Announcing Amazon S3 Files, making S3 buckets accessible as file systems

**カテゴリ:** What's New
**公開日:** 2026-04-07T14:15:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-s3-files/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-s3-files/)

---

## 要約

Amazon S3 Filesは、S3バケット内のデータにファイルシステムとしてアクセス可能にする新機能であり、ファイルベースのアプリケーション、AIエージェント、MLチームがデータの複製やステージングなしにS3データを直接操作できるようになります。このサービス、データレイクと分析データをS3に保存し、複数のコンピュートリソースから共有アクセスが必要な組織に特に適しています。

## 詳細

**新機能について:**
- Amazon S3をファイルシステムとしてアクセス可能にするS3 Filesが発表されました。
- AWS計算リソースからS3バケット内のデータに直接ファイルシステムとしてアクセスできます。
- ファイルシステムの完全なセマンティクスと低レイテンシーパフォーマンスを提供します。
- データはS3に保存されたまま、ファイルシステムAPIとS3 APIの両方で同時にアクセス可能です。
- Amazon EFSを使用して構築されており、スケーラビリティと費用対効果を備えています。
- データの複製やステージングが不要になります。
- 複数テラバイト/秒の集約読み取りスループットを提供します。
- 34のAWSリージョンで一般提供されています。

**このアップデートが適している対象:**
- ファイルベースのアプリケーションやツールを使用する組織です。
- S3にデータレイクと分析データを保存している企業です。
- AIエージェントがメモリを永続化し、パイプライン間で状態を共有する必要がある場合です。
- MLチームがファイルの複製やステージングなしにデータ準備ワークロードを実行したい場合です。
- ファイルシステムとオブジェクトストレージ間のデータ同期の複雑性を排除したい組織です。
- 複数の計算リソースから同じS3データへの共有アクセスが必要な場合です。
