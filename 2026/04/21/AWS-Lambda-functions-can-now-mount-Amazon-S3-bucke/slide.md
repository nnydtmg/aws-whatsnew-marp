# AWS Lambda から Amazon S3 をファイルシステムとしてマウント
S3 Files 機能がサポート開始

**発表日: 2026年4月21日**

---

## 概要

### S3 Files が AWS Lambda で利用可能に

- **S3 バケットをネイティブなファイルシステムとしてマウント**
  - NFS プロトコルで標準的なファイル操作を実行
  - データをダウンロードすることなく直接アクセス

- **複数の Lambda 関数間でのシームレスなデータ共有**
  - カスタム同期ロジック不要
  - 同時アクセスに対応

- **AI・機械学習ワークロードに最適**
  - ステートフルなマルチステップワークフロー
  - エージェント間でのメモリ永続化

---

## 前提・背景

### 従来の課題

- **Lambda でのデータ処理の複雑性**
  - S3 オブジェクトをダウンロード → 処理 → アップロードの必要
  - 一時ストレージ容量制限（最大 10GB）
  - パフォーマンスオーバーヘッド

- **ステートフルワークフローの実装困難**
  - 複数 Lambda 関数間でのデータ共有が煩雑
  - 状態管理ロジックの実装要件

### Amazon S3 Files の登場

- **2026 年 4 月 7 日**: Amazon S3 Files 一般提供開始
- **EFS ベースの実装**: ファイルシステムのシンプルさを維持
- **S3 のスケーラビリティ**: オブジェクトストレージの利点を活かす

---

## 主な機能

### 1. ネイティブなファイルシステムマウント

- **NFS v4.2 対応**
  - 標準的なファイル操作が可能
  - コード変更不要（レガシーアプリケーション対応）

- **EC2・ECS・EKS・Lambda で利用**
  - 統一されたアクセス方法
  - 既存アプリケーションの活用

### 2. 複数関数間での共有ファイルシステム

- **同時接続対応**
  - 複数 Lambda 関数が同一 S3 Files に接続
  - 同期ロジック実装不要

- **透過的なデータ同期**
  - S3 が真実のソース
  - 完全性を損なわない同期機構

---

## 技術的詳細

### Amazon EFS 統合

- **基盤技術**: Amazon EFS
  - ファイルシステムのシンプルさ
  - S3 のスケーラビリティ・耐久性
  - コスト効率性

### 設定方法

- **AWS Management Console**: GUI で簡単設定
- **AWS CLI**: スクリプト化対応
- **AWS SDK**: プログラマティック制御
- **AWS CloudFormation**: IaC 対応
- **AWS Serverless Application Model (SAM)**: サーバーレス構成管理

  - 自動チェックポイント機能
  - マルチステップ AI ワークフロー

## 対応リージョン・サポート

### 利用可能地域

- **対応リージョン**: Lambda と S3 Files が利用可能なすべての AWS リージョン

### 追加料金

- **無料**: 標準的な Lambda および S3 料金のみ
  - S3 Files 固有の追加料金なし

### 対応 Lambda 環境

- **キャパシティプロバイダー非設定の Lambda 関数**
  - 標準的な Lambda 環境で利用可能
