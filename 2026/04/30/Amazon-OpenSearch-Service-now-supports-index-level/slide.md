# Amazon OpenSearch Service
インデックスレベルの暗号化をサポート

**発表日: 2026年4月30日**

---

## 概要

### Amazon OpenSearch Service の新機能

- **インデックスレベルの暗号化** に対応
- AWS KMS顧客管理キーを使用して、インデックスごとにデータを暗号化可能
- マルチテナント環境でのセキュリティ向上
- より細かなデータ分離制御が実現

### 主な特徴

- 同一ドメイン内の異なるインデックスに対して異なる暗号化キーを使用可能
- テナント固有の暗号化ポリシーを細かく制御可能
- 追加費用なしで利用可能

---

## 前提・背景

### セキュリティ要件の高まり

- マルチテナント環境での暗号化ポリシーの多様化
- コンプライアンス要件の厳格化
- テナント間でのデータ分離の重要性

### 従来のドメインレベル暗号化の課題

- 同一ドメイン内のすべてのインデックスに同じ暗号化キーを適用
- 複数テナントの要件に柔軟に対応できない
- コンプライアンス要件での細かい制御が困難

### OpenSearch の進化

- バージョン3.3以降で新機能をサポート
- エンタープライズ環境でのセキュリティ要件に対応

---

## 主な機能と改善

### 1. インデックスレベルの暗号化

- インデックスごとに個別のAWS KMSキーを指定可能
- インデックス間で暗号化されたデータを分離
- ドメインレベルの暗号化とは異なる、より細かい制御

### 2. 実装方法

- Amazon OpenSearch Service APIを使用してKMSキーを登録
- 暗号化されたインデックスを作成する際にキーARNを指定
- インデックス設定で簡単に暗号化ポリシーを適用

### 3. 対応環境

- **要件:** OpenSearch バージョン3.3以降
- **コスト:** 追加費用なし
- **リージョン:** 14のAWSリージョンで利用可能

---

## 効果・メリット

### セキュリティの向上

- **テナント間のデータ分離** が強化
- **コンプライアンス要件** への柔軟な対応
- 暗号化キーの独立管理が可能

### 運用の効率化

- 複数の部門やプロジェクトが同一ドメインを共有可能
- インデックスごとに独立した暗号化ポリシー適用
- より柔軟なセキュリティ管理が実現

### ビジネスメリット

- マルチテナント環境での信頼性向上
- コンプライアンス対応の迅速化
- セキュリティリスク低減

## ユースケース

### マルチテナント SaaS プラットフォーム

- 顧客ごとに異なる暗号化キーを使用
- テナント間でのデータ分離を強化
- HIPAA、PCI-DSS などの規制要件に対応

### 複数部門の共有インスタンス

- 各部門が独自の暗号化ポリシーを適用
- 機密情報の保護レベルを調整
- 運用コストを削減

## 利用開始ステップ

### 1. KMS キーの準備

```
AWS KMS 顧客管理キーを作成または選択
```

### 2. キーの登録

```
Amazon OpenSearch Service API を使用
```

### 3. インデックスの作成

```
インデックス設定でキー ARN を指定
```
