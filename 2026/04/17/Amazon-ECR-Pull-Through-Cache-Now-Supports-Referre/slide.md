# Amazon ECR プルスルーキャッシュ
OCI リファラーの自動検出と同期に対応

**発表日: 2026年4月17日**

---

## 概要

### 何が変わったか

- **OCI リファラーの自動検出**
  - 画像署名、SBOM、証明書の自動検出に対応
  
- **リファラーの自動同期**
  - アップストリームレジストリから関連アーティファクトを自動キャッシュ
  
- **シームレスな検証ワークフロー**
  - クライアント側の手動操作が不要
  - エンドツーエンドの検証が可能

---

## 前提・背景

### これまでの課題

- **リファラー情報の未取得**
  - プルスルーキャッシュ経由でリファラーをリストアップしても、アップストリームから取得されない
  
- **手動対応の必要性**
  - ユーザー側でアップストリームのリファラーを手動で取得・同期する必要があった
  
- **検証ワークフローの複雑化**
  - 複数レジストリ管理時の署名検証、SBOM 取得が手作業に

### 市場の動向

- **コンテナセキュリティの重要性**
  - イメージ署名、SBOM が DevSecOps 標準に
  
- **マルチレジストリ管理**
  - 複数リージョン・アカウント間でのイメージ管理が一般化

---

## 変更内容・新機能

### 1. リファラー API 対応

- プルスルーキャッシュで referrers API リクエストをサポート
- アップストリームに自動接続

### 2. 自動キャッシング機構

- **対応するリファラー種別**
  - 画像署名（Image Signatures）
  - ソフトウェア部品表（Software Bill of Materials）
  - アテステーション（Attestations）
  - その他 OCI リファラー

- リファラーアーティファクトがプライベートリポジトリに自動キャッシュ

### 3. エンドツーエンドワークフロー対応

- 署名検証ワークフロー
- SBOM 検出ワークフロー
- 証明書取得ワークフロー
