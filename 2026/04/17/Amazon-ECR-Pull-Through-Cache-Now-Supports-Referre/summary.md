# Amazon ECR Pull Through Cache Now Supports Referrer Discovery and Sync

**カテゴリ:** What's New
**公開日:** 2026-04-17
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-ecr-pull-through-cache-referrers/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-ecr-pull-through-cache-referrers/)

---

## 要約

Amazon ECR のプルスルーキャッシュ機能が OCI リファラー（画像署名、SBOM、証明書など）の自動検出と同期に対応し、クライアント側の手動操作なしにエンドツーエンドの検証ワークフローがシームレスに機能するようになりました。このアップデートは、セキュリティ要件を満たしながら複数のレジストリを管理する組織に特に有益です。

## 詳細

- Amazon ECR のプルスルーキャッシュ機能が、OCI リファラー（画像署名、SBOM、証明書など）の自動検出と同期に対応するようになりました。
- 従来は、プルスルーキャッシュルールが設定されたリポジトリでリファラーをリストアップしても、アップストリームリポジトリからのリファラーは返却されず、同期もされませんでした。
- 本アップデートにより、Amazon ECR のプルスルーキャッシュはリファラー API リクエスト時にアップストリームに接続し、関連するリファラーアーティファクトをプライベートリポジトリに自動的にキャッシュするようになります。
- このアップデートにより、エンドツーエンドの画像署名検証、SBOM 検出、および証明書取得ワークフローがプルスルーキャッシュリポジトリでシームレスに機能するようになり、クライアント側の回避策が不要になります。
- この機能は、Amazon ECR プルスルーキャッシュがサポートされているすべての AWS リージョンで利用可能です。
- このアップデートは、セキュリティとコンプライアンスの要件を満たす必要があり、複数のアップストリームレジストリからコンテナイメージを管理する組織に特に有益です。
