# Amazon S3 が 5つの新しいチェックサムアルゴリズムに対応

エンドツーエンドのデータ整合性検証がさらに強力に

**発表日: 2026年4月23日**

---

## 概要

### Amazon S3 のチェックサム機能が拡張

- **新たに5つのチェックサムアルゴリズムに対応**
  - MD5、XXHash3、XXHash64、XXHash128、SHA-512
  - 合計10個のアルゴリズムが利用可能に

- **データ整合性検証の強化**
  - 追加のツール不要でエンドツーエンドの検証が可能
  - オブジェクトメタデータにチェックサム値を自動保存

- **完全な地域カバレッジ**
  - 37のAWSリージョンで利用可能
  - 追加費用なし

---

## 前提・背景

### データ整合性の課題

- **大規模データ管理の複雑化**
  - クラウドストレージのボリューム拡大に伴うリスク増加
  - ネットワーク転送中のデータ破損リスク

- **規制要件への対応**
  - 金融機関や医療業界におけるデータ整合性の厳格な要件
  - 監査ログの需要増加

- **従来の課題**
  - 複数のツールを組み合わせる必要性
  - 既存システムとの統合の複雑さ

### AWS の進化

- **段階的なデータ整合性対応**
  - 以前: CRC32 だけをサポート
  - 現在: 10個のアルゴリズムから選択可能

---

## 新機能：5つの追加チェックサムアルゴリズム

### 対応アルゴリズム一覧

| アルゴリズム | 特性 | 用途 |
|-----------|------|------|
| **MD5** | 広く普及した標準ハッシュ | 汎用的な整合性確認 |
| **XXHash3** | 超高速計算 | 大規模ファイル向け |
| **XXHash64** | 64ビット高速ハッシュ | パフォーマンス重視 |
| **XXHash128** | 128ビット拡張 | より強い保証が必要な場合 |
| **SHA-512** | 強力な暗号学的ハッシュ | セキュリティ重視 |

---

## 機能の詳細

### 1. アップロード時の検証

```
ユーザーがチェックサム値を提供
        ↓
S3がデータを検証
        ↓
検証が成功すればオブジェクトを保存
        ↓
メタデータに値を記録
```

- **マルチパートアップロード対応**
  - パートレベルのチェックサムを個別に検証
  - 完了時に複合チェックサムを自動計算

### 2. 自動保護機能

- チェックサム値を提供しない場合
- S3 が自動的に **CRC64NVME** を計算・適用
- デフォルトでエンドツーエンド保護を実現

### S3 バッチオペレーション

- **既存オブジェクトのチェックサム計算**
  - データをダウンロードまたは復元することなく実行可能

  - 非同期処理で本番環境への影響を最小化

  - コスト効率的なバッチ処理

## 利用方法

### AWS CLI での使用例

```bash
# アップロード時にチェックサムを指定
aws s3api put-object \
  --bucket my-bucket \
  --key my-file.txt \
  --body my-file.txt \
  --checksum-algorithm SHA512

# 既存オブジェクトのチェックサムを確認
aws s3api head-object \
  --bucket my-bucket \
  --key my-file.txt
```

### AWS SDK による実装

- **Python、Java、Node.js** など主要言語対応
- **自動チェックサム計算**機能を搭載
- **最新バージョン** の SDK を推奨
