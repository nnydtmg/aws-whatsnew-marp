# Amazon Quick now supports permission verification for ACL-enabled knowledge bases

**カテゴリ:** What's New
**公開日:** 2026-04-23
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-quick-acl/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-quick-acl/)

---

## 要約

Amazon Quickは、ACL対応ナレッジベースに対して権限検証機能を新たに提供し、管理者が特定のユーザーのドキュメントアクセス権限を即座に確認できるようになりました。この機能により、アクセス問題のトラブルシューティングが簡素化され、機密ドキュメントの適切なアクセス制限を確認することができます。

## 詳細

- Amazon Quickは、ACL対応のナレッジベースに対して、権限検証機能をサポートするようになりました。
- この機能により、管理者は特定のユーザーが特定のドキュメントにアクセスできるかどうかを確認することができます。
- 同期レポートタブからアクセス詳細を表示し、ユーザーのメールアドレスを入力することで、ドキュメントへのアクセス可否を即座に確認できます。
- アクセス詳細パネルには、ドキュメントにアクセスできるすべてのユーザーとグループが表示され、適用されている権限の完全な可視性が得られます。
- 権限チェッカーは、ユーザーがアクセス可能、アクセス不可、またはACLが見つからないという3つの結果のいずれかを返します。
- この機能は、Amazon Quickが利用可能なすべてのAWSリージョンのACL対応ナレッジベースで利用可能です。
- データソース全体の権限継承を手動で追跡することなく、アクセス問題のトラブルシューティングが簡素化され、機密ドキュメントが適切に制限されていることを確認できます。
