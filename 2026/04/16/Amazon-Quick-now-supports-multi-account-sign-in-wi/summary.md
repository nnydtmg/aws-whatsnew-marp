# Amazon Quick now supports multi-account sign-in within the same browser

**カテゴリ:** What's New
**公開日:** 2026-04-16T12:00:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-quick-multi-account-sign-in/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-quick-multi-account-sign-in/)

---

## 要約

Amazon Quickのマルチセッション機能により、ユーザーは同一ブラウザ内で最大5つのアカウントに同時にアクセスでき、開発・テスト・本番環境などの複数環境を効率的に管理できるようになりました。この機能は、複数アカウントを運用し、アカウント間でリソース設定やインサイトを比較する必要があるお客様に特に有用です。

## 詳細

- Amazon Quickは、同一ブラウザ内で最大5つのAmazon Quickアカウントに同時にアクセスできるマルチセッション機能をサポートするようになりました。

- すべてのURLにAmazon Quickアカウント名が含まれるようになり、ユーザーはエージェント、スペース、フロー、リサーチレポート、ダッシュボード、およびその他のアセットにアクセスする際に、正しいアカウントに簡単にアクセスできるようになりました。

- 開発、テスト、本番環境などの異なる環境で複数のアカウントを使用している顧客は、単一のブラウザで複数のアカウントにサインインし、リソースを管理できるようになりました。

- Amazon Quickの右上メニューから別のアカウントへのサインインオプションを選択することで、別のアカウントにサインインできます。

- アカウント名を含まないグローバルURLにアクセスするユーザーには、ログイン済みのアカウントが事前に入力されたアカウント入力ページが表示され、目的のアカウントを選択できます。

- 現在のセッションをログアウトするか、すべてのセッションをログアウトするかのオプションが提供されます。

- このマルチアカウントサインイン機能は、すべてのサポートされているAmazon Quickリージョンで利用可能です。
