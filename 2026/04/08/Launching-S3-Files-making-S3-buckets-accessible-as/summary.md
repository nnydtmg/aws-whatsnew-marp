# Launching S3 Files, making S3 buckets accessible as file systems

**カテゴリ:** AWS Blog
**公開日:** 2026-04-07T19:18:32
**元記事:** [https://aws.amazon.com/blogs/aws/launching-s3-files-making-s3-buckets-accessible-as-file-systems/](https://aws.amazon.com/blogs/aws/launching-s3-files-making-s3-buckets-accessible-as-file-systems/)

---

## 要約

Amazon S3 Filesは、S3バケットをNFSファイルシステムとしてマウント可能にする新機能であり、EC2、ECS、EKS、Lambdaなどのコンピュートリソースから高性能で対話的なアクセスが実現されます。このアップデートは、AIエージェント、機械学習パイプライン、複数リソースでの協調的なデータ処理が必要な組織に特に適しており、S3の耐久性とコスト効率とファイルシステムの利便性を両立させることができます。

## 詳細

- Amazon S3 Filesは、AWS S3バケットをファイルシステムとしてアクセス可能にする新機能です。

- EC2、ECS、EKS、Lambda等のAWSコンピュートリソースから、S3データにNFSファイルシステムインターフェースでアクセスできます。

- ファイルシステム経由での変更は自動的にS3バケットに反映され、S3での変更はファイルシステムに数秒以内に表示されます。

- 複数のコンピュートリソースからの共有アクセスが可能であり、データ重複なしでクラスタ間のデータ共有ができます。

- 約1ミリ秒のレイテンシーで高性能なアクセスが実現され、インテリジェントなプリフェッチング機能も備わっています。

- このアップデートは、対話的でインタラクティブなワークロードを必要とする組織に適しています。

- 特にAIエージェント、機械学習トレーニングパイプライン、本番環境アプリケーションなど、複数リソースでデータを協調的に読み書きする必要がある場合に最適です。

- オンプレミスのNAS環境からの移行を検討している組織にも有用です。

- すべての商用AWSリージョンで本日から利用可能です。

- 料金はS3ファイルシステムに保存されたデータ、小ファイル読み取り、すべての書き込み操作、およびS3との同期リクエストに対して発生します。
