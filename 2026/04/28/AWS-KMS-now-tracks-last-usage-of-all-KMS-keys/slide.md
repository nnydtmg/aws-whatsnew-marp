# AWS KMS では全キーの最後の使用状況を追跡
キー使用状況の可視化とコンプライアンス強化

**発表日: 2026年4月27日**

---

## 概要

### AWS KMS のキー使用状況追跡機能

- **すべてのキーの最後の暗号化操作を自動追跡**
- **管理コンソールと API で使用状況を可視化**
- **セキュリティ管理者とコンプライアンスチームの効率化**
- **未使用キーの特定と自動クリーンアップに対応**

### 主な価値提案

- 手動ログ分析が不要に
- リアルタイムなキー使用状況の把握
- コンプライアンス要件への対応を簡素化

---

## 前提・背景

### 従来の課題

- キーが積極的に使用されているか確認が困難
- AWS CloudTrail のログをクエリして手動分析が必要
- 不要な鍵の特定に時間がかかる
- ポリシーベースの保護が限定的

### 市場背景

- セキュリティとコンプライアンスの重要性が増加
- 暗号化キー管理の複雑化に対応の必要性
- 自動化によるリスク低減の需要
- CloudTrail 統合による監査能力の向上要望

---

## 主な新機能

### 1. キー使用状況の自動追跡

- すべての KMS キーの**最後の暗号化操作**を記録
- タイムスタンプの自動保存
- 操作の種類と関連する CloudTrail イベント ID を表示

### 2. 管理コンソール/API での可視化

- AWS KMS 管理コンソールで使用状況を確認
- API を通じたプログラマティックアクセス対応
- リアルタイムなデータ取得が可能

### 3. ポリシーベースの保護

- **新しい条件キー**: `kms:TrailingDaysWithoutKeyUsage`
- 最近使用されたキーの誤削除を防止
- ポリシー定義による自動保護
