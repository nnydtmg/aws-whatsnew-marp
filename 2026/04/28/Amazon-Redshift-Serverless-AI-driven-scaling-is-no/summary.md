# Amazon Redshift Serverless AI-driven scaling is now the default for new workgroups

**カテゴリ:** What's New
**公開日:** 2026-04-27T22:03:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-redshift-serverless-ai-driven-scaling-default/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-redshift-serverless-ai-driven-scaling-default/)

---

## 要約

Amazon Redshift Serverlessは、AI駆動スケーリングと最適化をすべての新しいワークグループのデフォルト機能として提供するようになりました。これにより、機械学習を活用した自動リソース調整と、サポート対象のBase RPU範囲の拡大（8～512 RPU）により、手動チューニングなしでより優れた価格性能を実現できるようになっています。

## 詳細

### コア機能

- Amazon Redshift Serverless のAI駆動スケーリングが、新しいワークグループのデフォルト機能になりました。
- 機械学習を使用してコンピュート需要を予測し、クエリがキューイングされる前に自動的にリソースを調整します。
- 手動チューニングなしで、より良い価格性能を実現できます。

### RPU範囲の拡大

- サポート対象のBase RPU範囲が8～512 RPUに拡大され、以前の32～512 RPUから下限が引き下げられました。
- AI駆動スケーリングにおけるエントリーコストが削減されました。
- 小規模ワークロードから大規模分析まで、幅広いユースケースに対応

### 自動最適化機能

- Amazon Redshiftはワークロードパターンを監視し、クエリの複雑さ、データボリューム、予想されるデータスキャンサイズに基づいてコンピュートリソースを自動調整します。
- 価格性能スライダーを使用して、コスト、パフォーマンス、またはその両方のバランスを優先するかを選択できます。
- 自動マテリアライズドビューと自動テーブル設計最適化などの追加最適化が適用されます。

### 管理と設定

- AWS Management ConsoleまたはAmazon Redshift APIを使用して価格性能ターゲットを設定できます。
- ワークグループ作成後にターゲットを変更することも可能です。
- この機能はAmazon Redshift Serverlessが利用可能なすべてのAWSリージョンで利用できます。

## ビジネスインパクト

### コスト削減

- 最小 8 RPU からのエントリーで小規模ワークロードでも経済的に利用可能
- AI駆動スケーリングにより不必要なリソース確保を排除
- Amazon内部テストで最大10倍の価格性能向上を実証

### 運用効率化

- 手動チューニング作業が不要
- デフォルト有効で追加設定が最小限
- 既存ワークグループでも手動有効化可能

### 技術的メリット

- 機械学習による予測的なリソース管理
- クエリがキューイングされる前のプロアクティブスケーリング
- ワークロード特性に応じた柔軟な最適化

## 対応範囲

- Amazon Redshift Serverless が利用可能なすべての AWS リージョンで利用可能
- 新規ワークグループ作成時にデフォルト有効
- 既存ワークグループへの手動適用も可能
