# AWS Glue now supports OAuth 2.0 for Snowflake connectivity

**カテゴリ:** What's New
**公開日:** 2026-04-21T16:00:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/aws-glue-oauth-snowflake-connectivity/](https://aws.amazon.com/about-aws/whats-new/2026/04/aws-glue-oauth-snowflake-connectivity/)

---

## 要約

AWS GlueのSnowflakeコネクタがOAuth 2.0認証をサポートするようになり、ユーザー認証情報を共有することなくセキュアなトークンベースの認証でSnowflakeデータへのアクセスが可能になりました。これにより、エンタープライズ企業はセキュリティコンプライアンスを維持しながら、データ統合パイプラインの構築がより容易になります。

## 詳細

- AWS Glueは、Snowflakeへのネイティブ接続にOAuth 2.0認証をサポートするようになりました。

- ユーザー認証情報を共有することなく、Snowflakeのデータを読み書きできるようになりました。

- 従来は永続的な認証情報または秘密鍵を使用する必要がありましたが、セキュアな一時的なトークンベースの認証に変更されました。

- 認証情報管理を完全に排除でき、セキュリティコンプライアンスの維持がより容易になりました。

- ユーザーとアプリケーションに対して、きめ細かいアクセス制御を定義することが可能です。

- トークンベースの認証により、組織全体のデータアクセスパターンの追跡と監視が改善されました。

- この機能は、AWS Glueが利用可能なすべてのAWSコマーシャルリージョンで提供されています。
