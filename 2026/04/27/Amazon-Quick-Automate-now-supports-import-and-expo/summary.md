# Amazon Quick Automate now supports import and export of automations across AWS accounts

**カテゴリ:** What's New
**公開日:** 2026-04-20T20:12:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/quick-automate-import-export/](https://aws.amazon.com/about-aws/whats-new/2026/04/quick-automate-import-export/)

---

## 要約

Amazon Quick Automateの新機能により、オートメーションをAWSアカウントとリージョン間で安全かつ効率的に移行できるようになりました。このアップデートは、マルチ環境デプロイメントを管理するDevOpsチームと組織にとって、開発から本番環境への昇格と災害復旧シナリオに対応するための重要な機能です。

## 詳細

- Amazon Quick Automateは、AWSアカウント、AWSリージョン、オートメーショングループ間でオートメーションバージョンを移行できるようになりました。

- AWS KMS暗号化を使用して、ワークフロー、ランタイム構成、プロセスステップメタデータを安全に暗号化されたリンクにパッケージ化します。

- エクスポートリンクは12時間有効で、複数回の再利用が可能であり、手動での再作成と比較して大幅な時間節約が実現できます。

- 開発環境から本番環境へのオートメーション昇格、複数のAWSリージョンへのデプロイ、チーム間での実証済みワークフロー共有が主な使用例です。

- 米国東部（バージニア北部）、米国西部（オレゴン）、ヨーロッパ（ダブリン）、ヨーロッパ（ロンドン）、ヨーロッパ（フランクフルト）、アジアパシフィック（東京）、アジアパシフィック（シドニー）を含むすべてのAWSリージョンで利用可能です。

- アクションコネクタ、認証情報、人間が関与するタスクキュー構成などの依存関係は移行バンドルに含まれず、宛先環境で別途構成する必要があります。
