# Amazon CloudWatch RUM now supports an improved App Monitors overview with health, SLO, and tracing status

**カテゴリ:** What's New
**公開日:** 2026-04-27T19:32:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-cloudwatch-rum-app-monitors/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-cloudwatch-rum-app-monitors/)

---

## 要約

Amazon CloudWatch RUMの改善されたApp Monitors概要ページにより、DevOpsおよびSREチームは、フロート全体のヘルスステータス、SLO違反、トレーシングカバレッジをシングルページで確認でき、各モニターを個別にクリックすることなく効率的にトリアージと監視ができるようになりました。このアップデートは、複数のアプリケーションモニターを管理し、可視性とSLO準拠を重視する組織にとって特に有用です。

## 詳細

### 改善内容

- Amazon CloudWatch RUMが改善されたApp Monitorsの概要ページをサポートするようになりました。
- ヘルスステータス、SLO違反、分散トレーシングカバレッジがシングルページに表示されるようになりました。
- 4つのサマリーカード（注意が必要、悪化傾向、セットアップとカバレッジ、SLOとアラーム）でモニターをグループ化しています。

### 新機能

#### クイックフィルター機能
- プラットフォーム、ヘルスステータス、SLIステータス、トレーシング状態で特定のアプリモニターに絞り込むことができます。

#### 詳細テーブル表示
- App Monitorsテーブルの各行には以下が表示されます：
  - セッションボリューム
  - SLIステータス
  - ヘルスステータス
  - 主な問題タイプ
  - トレンド方向
  - AWS X-Rayのトレースへの直接リンク
  - CloudWatch Application Signalsのリンク済みサービスヘルス
  - 最後のイベント受信時刻

#### サイドパネル
- 選択可能なサイドパネルで以下の追加情報を確認できます：
  - 相関セッション
  - アプリモニターヘルス
  - SLOとアラーム詳細

### 提供状況

- CloudWatch RUMが利用可能なすべてのAWSコマーシャルリージョンで追加コストなしで利用可能です。

## 背景

複数のアプリケーションモニターを管理する組織では、ヘルスステータス、SLO準拠状況、分散トレーシング情報を一画面で確認することが重要です。従来は各モニターを個別に確認が必要だったため、効率的な監視・トリアージが課題でした。

## メリット

### 運用効率
- 複数画面の移動が不要
- 素早いリスク特定と優先順位付け
- MTTR（平均修復時間）の削減

### 可視性
- SLO準拠状況のリアルタイム把握
- パフォーマンストレンドの一元表示
- 分散トレーシングカバレッジの透視

## ユースケース

- 複数アプリケーションモニターを運用する企業
- SLO達成を重視する組織
- DevOps/SRE文化が浸透した企業
- グローバル規模でアプリケーション運用を行う組織

## 関連サービス

- **Amazon CloudWatch Application Signals**: SLO機能の統合（2026年3月更新）
- **AWS X-Ray**: 分散トレーシング
- **CloudWatch RUM**: リアルユーザーモニタリング
