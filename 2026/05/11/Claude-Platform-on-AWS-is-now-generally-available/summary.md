# Claude Platform が AWS で一般提供開始

Claude Platform on AWS is now generally available

**カテゴリ:** What's New
**公開日:** 2026-05-11T14:00:00
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/05/claude-platform-aws/)

このページでは、AWS What's Newで発表された「Claude Platform on AWS is now generally available」の内容を日本語で要約し、スライド形式で確認できます。

---

## 要約

AWS上でAnthropicのネイティブClaudeプラットフォームが一般提供開始され、既存のAWSアカウントから直接アクセスできるようになりました。このサービスは、統合請求とセキュリティ管理を求める開発チームおよび企業向けに設計されており、複数のリージョンで利用可能です。

## このアップデートで何が変わったか

**新機能について**
- AWS上でAnthropicのネイティブClaudeプラットフォームに直接アクセスできるサービスが一般提供開始されました。
- 既存のAWSアカウントを通じて、API、コンソール、ベータ版機能にアクセスできるようになりました。
- Claude Managed Agents（ベータ版）、アドバイザー戦略（ベータ版）、ウェブ検索、ウェブ取得、コード実行、Files API（ベータ版）、Skills（ベータ版）、MCPコネクタ（ベータ版）、プロンプトキャッシング、引用、バッチ処理、Claudeコンソールなどの機能が利用できます。
- 既存のIAM認証情報、アクセス制御、統合されたAWSの請求、CloudTrailの監査ログが使用できます。
- 複数のリージョン（北米、南米、ヨーロッパ、アジア太平洋地域）で利用可能です。

## 対象ユーザー

**このアップデートが適している対象**
- 別々のアカウント管理や請求追跡を行いたくない開発チームです。
- Anthropicのネイティブなプラットフォーム開発体験にアクセスしたい企業です。
- 特定のリージョンデータレジデンス要件がない組織です。
- 既存のAWSセキュリティとアクセス制御を活用したい開発チームです。

## メリットと効果

### 運用効率化
- 1つのAWSアカウントで統合管理
- 複数プラットフォーム間の管理コスト削減
- 統合請求による請求処理の自動化
- チャージバック処理の簡素化

### セキュリティとコンプライアンス
- IAM ポリシーの統一的な適用
- CloudTrail による監査ログの完全記録
- AWSセキュリティ境界との統合
- Anthropic との協調による高レベルセキュリティ

### 開発生産性向上
- AWS SDK との連携
- Claude最新機能への即座のアクセス
- Anthropicネイティブの開発体験
