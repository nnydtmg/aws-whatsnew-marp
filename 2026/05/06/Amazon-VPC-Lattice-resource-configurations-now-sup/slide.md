# Amazon VPC Lattice
プライベートドメイン名ターゲット対応

**発表日: 2026年5月4日**

---

## 概要

### VPC Lattice の新機能

- **プライベートドメイン名ターゲット対応**: プライベートFQDNをリソース設定として定義可能
- **クロスアカウント共有**: 他のアカウントと安全に共有できるように
- **DNS解決の柔軟性**: 公開DNS エントリ不要でプライベートDNSサーバーに対応

### 実装のポイント

- リソースゲートウェイの「Resource Config DNS Resolution」を「IN_VPC」に設定
- VPCのDNS設定を活用したトラフィックルーティング
- 追加コストなしで利用可能

---

## 前提・背景

### これまでの課題

- **制限事項**: 公開的に解決可能なドメイン名ターゲットのみ共有可能
- **非対応**: プライベートDNSサーバー利用環境でのFQDN共有ができなかった
- **セキュリティ懸念**: 公開DNS エントリが必要だった

### VPC Lattice の進化

- エンタープライズのプライベートネットワークに対応
- 複雑なマルチアカウント構成に対応
- セキュリティ要件の厳格化に対応

---

## 主な変更内容

### 1. プライベートFQDN対応

- **新機能**: プライベートドメイン名をリソース設定に定義
- **効果**: プライベートDNSサーバーを使用する環境で機能が利用可能
- **共有**: 他のアカウントとの安全な共有が実現

### 2. DNS解決の最適化

- **設定**: Resource Config DNS Resolution を「IN_VPC」に設定
- **メカニズム**: VPCのDNS設定を使用してFQDNを解決
- **ルーティング**: 公開DNS エントリなしで正しいバックエンドへトラフィック転送

### 3. 設定方法

- AWS Management Console
- AWS CLI
- AWS SDKs
- AWS APIs

- ✅ **クロスアカウント安全共有**: プライベートネットワーク内での通信

### 運用面

- ✅ **追加コストなし**: 既存VPC Lattice価格で利用可能

### ビジネス面

- ✅ **複数チーム間連携**: 安全にリソース共有可能
- ✅ **ハイブリッド構成対応**: オンプレミスDNS統合対応
- ✅ **スケーラビリティ**: 大規模環境での展開が容易

## ユースケース

### 1. マルチアカウント環境でのプライベートサービス共有

**シナリオ**:
- 複数の部門が異なるAWSアカウント運用
- プライベートドメインでバックエンド共有が必要
- VPC Latticeでクロスアカウントアクセス実現

**メリット**: 公開DNS汚染なし、セキュアな共有

### 2. ハイブリッドクラウド環境

**シナリオ**:
- オンプレミスの社内DNS使用
- AWS とオンプレ間のサービス共有
- プライベートFQDNでの統一的なアクセス

**メリット**: DNS解決の一元化、シームレスな連携

### 3. エンタープライズセキュリティ要件

**シナリオ**:
- すべてのリソースがプライベートIPのみ
- 公開DNS使用不可ポリシー
- 部門間の安全なサービス共有

**メリット**: ポリシー準拠、監査対応
