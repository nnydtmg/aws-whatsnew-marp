# AWS Security Hubが未使用アクセスの識別リスク検出に対応

**発表日: 2026年5月20日**

---

## 概要

### 統一されたコンソールで識別リスクを一元管理

- **未使用アクセスの検出**: IAMアクセス権限、ロール、認証情報の未使用状態を自動検出
- **一元管理**: 脅威、露出、体勢（ポスチャー）と共に識別リスクを統一コンソールで管理
- **自動化**: Security Hub有効化時にサービスリンクIAM Access Analyzerが自動作成
- **追加費用なし**: これらの機能はSecurity Hub Essentialsに含まれます

---

## 前提・背景

### セキュリティ運用の課題

#### 従来の問題点
- 複数のツール間を切り替えながら運用
- 未使用アクセスと実際のリスク露出の関連性が不明確
- 数百のアカウント規模では管理が複雑化
- 組織全体の識別リスクの全体像を把握が困難

#### 市場トレンド
- **Identity is the new perimeter**: アイデンティティがセキュリティの最前線
- IAMセキュリティベストプラクティスの重要性が高まる
- 90日以上未使用の認証情報は重大な脅威
- 最小権限ポリシーへの移行が必須要件

---

## 主な機能・変更内容

### 1. 未使用アクセスの検出

- 過去**90日間の実際のアクセスアクティビティ**に基づいて評価
- IAMプリンシパル（ユーザー、ロール、認証情報）を自動分析
- 統一コンソール内で脅威や露出と共に表示
- 実際の使用パターンに基づく検出で誤検知を最小化

### 2. 統一ダッシュボード

- 脅威、露出、体勢、**識別リスク**の4つのカテゴリを一箇所に集約
- セキュリティチームが優先順位付けしやすい構成
- 複数ツール間の切り替え作業を削減

### 3. 最小権限ポリシー生成

- 実際の使用パターンに基づいた推奨ポリシーを**オンデマンド生成**
- IAM権限の段階的な調整が可能
- 攻撃面を効果的に削減

---

## 技術詳細

### サービス連携

```
Security Hub（有効化）
        ↓
サービスリンクIAM Access Analyzer
が各メンバーアカウントに自動作成
        ↓
追加設定不要で動作開始
```

### 検出プロセス

1. **アクティビティ分析**: CloudTrailデータから90日間のアクセス実績を収集
2. **プリンシパル評価**: IAMプリンシパルごとに使用状況を分析
3. **リスク判定**: 未使用と判定されたアクセス権限をリスクとして抽出
4. **コンテキスト相関**: 露出情報と組み合わせてリスク優先度を算出

## メリット・効果

### セキュリティ運用の効率化

- **工数削減**: 複数ツール管理の必要性がなくなる
- **迅速な対応**: 優先度順に表示されるため対応順序が明確
- **攻撃面縮減**: 不要なアクセス権限を迅速に削減

### コスト削減

- **追加コスト無し**: Security Hub Essentialsに含まれる
- **ポリシー最適化**: 権限の無駄を可視化し削減
- **ライセンス効率**: 単一プラットフォームでまとめて管理

### リスク低減

- **360°のセキュリティ可視性**: 脅威、露出、体勢、識別リスクを一覧
- **未使用アクセスの排除**: セキュリティグループの権限削減
- **最小権限への移行**: 実績ベースのポリシー推奨で安全な権限設定

## ユースケース

### ユースケース1: 規制対応

- **HIPAA, SOC2等** の規制要件に対応
- 未使用アクセスの即時排除による監査対応
- 権限ポリシーの生成記録で監査証跡を提供

### ユースケース2: セキュリティインシデント対応

- 侵害されたプリンシパルの権限範囲を迅速に把握
- 最小権限を基に横展開リスクを評価
- 緊急時の権限削減を効率的に実施

### ユースケース3: 定期的なアクセス監査

- 四半期ごとのアクセス権限見直し自動化
- 部門異動に伴う権限変更の自動検出
- 権限クリーニング施策の実施

---

## 利用開始方法

### ステップ1: Security Hubの有効化
- AWS Consoleから Security Hubを有効化
- または CloudFormation / CDK で自動デプロイ

### ステップ2: 自動セットアップ
- サービスリンクIAM Access Analyzerが各メンバーアカウントに自動作成
- 追加の手動設定は不要

### ステップ3: 検出結果の確認
- Security Hubコンソールで未使用アクセス関連の検出結果を表示
- ポリシー推奨事項をレビュー

### ステップ4: ポリシー生成・適用
- オンデマンドで最小権限ポリシーを生成
- 検証後にIAM権限を更新
