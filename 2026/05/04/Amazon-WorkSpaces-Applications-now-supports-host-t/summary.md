# Amazon WorkSpaces Applications now supports host-to-client URL redirection

**カテゴリ:** What's New
**公開日:** 2026-05-04
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-workspaces-applications-url/](https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-workspaces-applications-url/)

---

## 要約

Amazon WorkSpaces ApplicationsのホストからクライアントへのURL リダイレクション機能により、ストリーミングセッション内のURLをユーザーのローカルブラウザで自動的に起動し、インフラストラクチャの負荷を軽減することができるようになりました。この機能は、セキュリティを保ちながらコスト効率を改善したい組織に特に適しています。

## 詳細

- Amazon WorkSpaces Applicationsが、ホストからクライアントへのURL リダイレクション機能をサポートするようになりました。
- ストリーミングセッションからのURLが、ユーザーのローカルブラウザで自動的に起動されます。
- 管理者はAWS Management Consoleを通じて、許可および拒否するURLパターンを設定できます。
- 機密性の高いアプリケーションはストリーミング環境内に保持しながら、ビデオストリーミングなどのリソース集約的なコンテンツをローカルデバイスにオフロードできます。
- ストリーミングインフラストラクチャの負荷を軽減し、帯域幅集約的なワークロードをローカルデバイスにシフトさせることで、インフラストラクチャコストを削減できます。
- ブラウザナビゲーションおよびMicrosoft Wordなどのアプリケーション内の埋め込みリンクで機能します。
- ChromeおよびEdgeウェブブラウザをサポートしています。
- 複数のAWSリージョンで利用可能です。
- このアップデートは、ストリーミングインフラストラクチャのコスト最適化を必要とする組織に適しています。
- セキュリティを維持しながらユーザーエクスペリエンスを向上させたい企業に有益です。
