# Amazon Quick introduces Dataset Q&A for conversational analytics against enterprise data

**カテゴリ:** What's New
**公開日:** 2026-05-04T16:55:00
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-quick-dataset-qa/](https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-quick-dataset-qa/)

---

## 要約

Amazon Quickの新しいDataset Q&A機能により、ユーザーは自然言語でエンタープライズデータに対して直接質問でき、テキストからSQLへの変換エージェントが正確なSQLを生成して、複数のデータソースから意味のある洞察を得ることができるようになりました。この機能は、データセットへのアクセス権を持つすべてのユーザーに対して、行レベルおよび列レベルのセキュリティポリシーを尊重しながら、データ探索と分析の民主化を実現します。

## 詳細

### 新機能について

- Amazon Quickが Dataset Q&A という会話型分析機能を導入しました
- ユーザーが自然言語で企業データに対して直接質問できるようになりました
- テキストからSQLへの変換エージェントにより、ユーザーの質問を解釈し、適切なデータを特定し、正確なSQLを生成します
- Amazon Redshift、Amazon Athena、Aurora PostgreSQL、Amazon S3のApache Icebergテーブルなど、複数のデータソースに対応しています
- データ所有者がカスタム指示、ビジネス定義、フィールド説明をメタデータとして追加できます
- ナレッジグラフを活用してデータ資産の意味と関係性を捉えています
- トレンド分析、時系列比較、ランキング、複数条件の分析クエリなど、幅広い質問タイプに対応しています
- Explain機能により、ユーザーが生成されたSQLの推論プロセスを検証できます
- 行レベルセキュリティと列レベルセキュリティのガバナンスルールを尊重しています

### 対象ユーザーについて

- データセットへのアクセス権を持つすべてのユーザーが対象です
- 自然言語を使用してデータを探索したいビジネスユーザーに適しています
- データの意味と関係性を理解し、正確な分析結果を必要とするエンタープライズユーザーに有用です
- データガバナンスとセキュリティを重視する組織に適しています
