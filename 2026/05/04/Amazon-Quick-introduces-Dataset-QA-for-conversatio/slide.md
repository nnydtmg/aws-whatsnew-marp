# Amazon Quick Dataset Q&A
会話型分析でエンタープライズデータを民主化

**発表日: 2026年5月4日**

---

## 概要

### Amazon Quick Dataset Q&A の3つの主要特徴

- **自然言語クエリング**: ユーザーは SQL 不要で、自然言語でエンタープライズデータに直接質問可能
- **テキストからSQLへの自動変換**: AI エージェントがユーザーの質問を解釈し、正確な SQL を自動生成
- **完全なセキュリティ準拠**: 行レベル・列レベルセキュリティを自動的に尊重しながら結果を返却

### 利用対象

- データセットへのアクセス権を持つすべてのユーザー
- SQL スキルを持たないビジネスユーザー
- データ分析の自動化と民主化が必要な組織

---

## 前提・背景

### 市場の課題

- **SQL スキルの欠落**: 大多数のビジネスユーザーが SQL を習得していない
- **分析ボトルネック**: ビジネスユーザーはデータ分析のため、ずっとアナリストに依存
- **複雑なクエリ作成**: 多表結合、時系列計算、階層的集計には高度なスキルが必要

### 既存の課題

- Dashboard Q&A のみでは、データセット全体への直接アクセスが限定的
- テキスト-SQL 生成技術の精度が不十分だった従来の実装

### Amazon Quick の進化

- 2026年5月に Dataset Q&A を一般提供開始
- ナレッジグラフを活用した意味的なデータ理解を実装

---

## 主な機能・改善

### 1. テキスト-SQL エージェント

- ユーザーの自然言語質問を解釈
- 適切なデータソースを特定
- エンジン・方言に対応した最適な SQL を自動生成
- 結果を数秒で返却

### 2. 対応するデータソース

| データソース | 説明 |
|-----------|------|
| **Amazon Redshift** | DWH としての主要なデータソース |
| **Amazon Athena** | S3 上の構造化データへのクエリ |
| **Aurora PostgreSQL** | リレーショナルデータベース |
| **Apache Iceberg (S3)** | オープンテーブルフォーマット対応 |
| **SPICE** | Amazon Quick の高速メモリキャッシュ |

---

## 主な機能・改善（続き）

### 3. メタデータ & ナレッジグラフ

- **データ所有者による機能強化**:
  - カスタム指示の追加
  - ビジネス定義の設定
  - フィールド説明の記入

- **ナレッジグラフによる理解**:
  - データ資産間の意味的な関係を捉える
  - より正確なデータセット選択と SQL 生成

### 4. 対応する質問タイプ

- トレンド分析（時間軸での変化）
- 時系列比較（複数期間の比較）
- ランキング分析
- 複数条件の分析クエリ
- 開放型の探索的質問

---

## 主な機能・改善（続き）

### 5. Explain 機能

- ユーザーが生成された SQL を検証可能
- 推論ロジックを段階的に確認
- 生成 SQL が質問を正しく解釈しているか検証
- 結果に対する信頼性を向上

### 6. ガバナンス & セキュリティ

- **行レベルセキュリティ (RLS)**: ユーザーごとに見えるデータを制限
- **列レベルセキュリティ (CLS)**: ユーザーごとに見える列を制限
- **自動尊重**: 追加設定不要で自動的にセキュリティルールを適用

---

## 効果・メリット

### ビジネス効果

- **分析の民主化**: SQL 未習得ユーザーもデータアクセス可能
- **アナリスト負荷低減**: 日常的なレポートリクエストの自動化により 71% の依存度削減が実現（参考事例）
- **意思決定スピード向上**: 秒単位でのインサイト取得

### 運用効果

- **セキュリティの簡素化**: RLS/CLS ルールを 1 度設定すれば自動適用
- **メタデータの価値向上**: ナレッジグラフ活用で組織的なデータ理解を深化
- **監査対応**: 生成 SQL をトレース可能で、コンプライアンス要件に対応

## ユースケース

### 1. 営業チーム向け自動レポート

- 営業担当者が「先月の地域別売上」と自然言語で質問
- 自動的に適切な SQL が生成され、即座に結果を表示
- 営業分析チームへの依存を大幅削減

### 2. マーケティング分析の迅速化

- マーケターが「Q1 キャンペーン ROI の推移」を質問
- ナレッジグラフが関連データを自動抽出
- ビジネス定義に沿った分析結果を即時取得

### 3. 財務部門の予測分析

- CFO が「部門別コスト削減の機会」を探索
- 複雑なマルチテーブル結合クエリを自動生成
- セキュリティルールを尊重しながら結果提供

### 4. データガバナンスの強化

- CLS でセンシティブ列（給与など）を制限
- RLS で従業員は自分のデータのみアクセス可能
- データ探索の自由度とセキュリティの両立

## まとめ

### Amazon Quick Dataset Q&A の位置づけ

- **データ民主化の加速**: SQL 不要の自然言語クエリで全従業員がデータアクセス可能
- **エンタープライズ対応**: RLS/CLS によるセキュリティとガバナンスを完全準拠
- **AI エージェント技術の応用**: テキスト-SQL 変換で複雑な分析クエリも自動生成

### 推奨される次のステップ

1. 既存の Amazon Quick 環境で Dataset Q&A の試用を開始
2. ビジネス定義とメタデータをナレッジグラフに追加
3. パイロットプロジェクトで ROI 検証（アナリスト負荷削減など）
4. 本番環境への段階的展開と組織全体への展開
