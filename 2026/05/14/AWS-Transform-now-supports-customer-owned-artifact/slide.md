# AWS Transform がカスタマー所有のアーティファクトストアに対応

**発表日: 2026年5月14日**

---

## 概要

- **AWS Transform** がカスタマー所有の Amazon S3 バケットをアーティファクトストアとしてサポート開始
- 顧客はデータの保存場所とセキュリティ対策を完全にコントロール可能
- データソブリンティーとコンプライアンス要件に対応

---

## 前提・背景

### 規制業界の課題

- データが自社または指定地域に保存されることが必須要件
- コンプライアンス規制に対応する必要性の増加
- データ主権（Data Sovereignty）の重要性

### AWS Transform とは

- エンタープライズアプリケーション移行・現代化ツール
- マイグレーション実務者が効率的に大規模移行を実行
- アーティファクト管理が重要な機能

---

## 主な機能・変更内容

### 1. カスタマー所有 S3 バケット対応

- 顧客が所有する Amazon S3 バケットを指定可能
- AWS Transform がバケット内のアーティファクトを直接利用

### 2. 暗号化とアクセス制御

- AWS KMS キーを使用したアーティファクト暗号化
- AWS アカウント内でのアクセスポリシー管理

### 3. マルチアカウント対応

- 複数の AWS アカウント全体でアーティファクトストレージを一元化
- ファイルを直接バケットにアップロード可能

---

## 利用のメリット

- **完全な管理権**: データの保存場所と暗号化を自社でコントロール
- **コンプライアンス対応**: 規制業界の厳しい要件に対応
- **運用効率化**: マイグレーション作業の効率化
- **スケーラビリティ**: 複数アカウント間でのアーティファクト共有

---

## 対象となる組織

- 規制業界（金融、医療、政府機関など）
- データソブリンティー要件が厳しい企業
- 複数の AWS アカウントを管理する大規模組織
- 自社データストレージに対する完全なコントロールが必要な顧客

---

## 提供地域・可用性

- AWS Transform が提供されているすべての AWS リージョンで利用可能
- グローバル対応

---

## まとめ

### 主な特徴

- 顧客所有の Amazon S3 バケット対応により、完全なデータ管理が実現
- AWS KMS 暗号化で強力なセキュリティを実装
- マルチアカウント環境での一元管理が可能

### 次のステップ

1. 既存のマイグレーションプロジェクトを評価
2. カスタマー所有 S3 バケット環境の設定
3. AWS Transform の機能を活用した効率的な移行実行
