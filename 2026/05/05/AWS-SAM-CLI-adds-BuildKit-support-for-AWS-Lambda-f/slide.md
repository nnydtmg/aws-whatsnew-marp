# AWS SAM CLI が BuildKit をサポート

Lambda コンテナイメージのビルド高速化と効率化

**発表日: 2026年5月5日**

---

## 概要

### AWS SAM CLI に BuildKit 統合

- **ビルド高速化**: マルチステージビルドと改善されたキャッシング機能
- **効率的なイメージ作成**: 開発用依存関係を除外した最小サイズのイメージ
- **クロスアーキテクチャ対応**: x86_64 と arm64（AWS Graviton2）の両者に対応
- **セキュリティ強化**: Docker シークレットで認証情報を保護

---

## 前提・背景

### コンテナイメージパッケージングの課題

- Lambda 関数をコンテナイメージとしてパッケージング利用が拡大
- 従来のビルド方法では本番環境最適化に制限あり
- 開発環境と本番環境のビルド要件が異なる
- マルチアーキテクチャ対応の需要増加

### BuildKit の登場

- Docker 公式の次世代ビルドエンジン
- より高度なビルド機能を提供
- マルチステージビルド、並列処理対応

---

## 主な機能

### 1. マルチステージビルド

- 開発用依存関係を除外
- 最終イメージサイズを大幅削減
- ビルド効率の向上

### 2. 改善されたキャッシング

- ビルドレイヤーの効率的な再利用
- リビルド時間を短縮
- CI/CD パイプラインの最適化

### 3. クロスアーキテクチャビルド

- x86_64 と arm64 に対応
- 同じマシンから複数アーキテクチャへビルド可能
- AWS Graviton2 対応の強化

---

## 主な機能（続き）

### 4. ビルドステップの並列化

- 複数のビルドステップを同時実行
- 全体のビルド時間削減
- リソース効率の向上

### 5. Docker シークレット管理

- ビルド時にシークレットを安全に使用
- 認証情報をイメージレイヤーから除外
- API キー、認証トークンの保護

---

## ご利用方法

### 必要な環境

- **SAM CLI バージョン**: 1.159.0 以降
- **対応コンテナランタイム**: Docker または Finch

### 利用開始

\`\`\`bash
# SAM CLI をアップデート
sam update

# BuildKit フラグを指定してビルド
sam build --use-buildkit
\`\`\`

## 効果・メリット

### ビルドプロセスの最適化

- **ビルド時間短縮**: キャッシング効率化で再ビルド時間が大幅削減
- **イメージサイズ削減**: マルチステージビルドで最終イメージが軽量化
- **開発生産性向上**: ローカル開発での迅速なビルド・テスト

### 本番環境への対応

- **マルチアーキテクチャ対応**: ARM ベースマシンでの実行最適化
- **セキュリティ強化**: 機密情報の安全な取り扱い
- **エンタープライズ対応**: 複雑なビルド要件への対応

## 注意点・制限事項

### 互換性

- Docker と Finch の両方で動作
- SAM CLI 1.159.0 以降が必須
- 既存の SAM プロジェクトは互換性を保持

### 今後の展開

- BuildKit の今後の機能追加に対応予定
- 他のコンテナランタイムへの拡大検討
- AWS Lambda 側の最適化との連携
