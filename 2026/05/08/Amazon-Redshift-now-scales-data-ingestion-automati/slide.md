# Amazon Redshift
データ取り込みの同時実行スケーリング対応

**発表日: 2026年5月7日**

---

## 概要

### Amazon S3 からの COPY クエリに対応

- **同時実行スケーリング**が COPY クエリをサポート開始
- 取り込み速度**とクエリパフォーマンスの両立**を実現
- バッチワークロードの**自動スケーリング**に対応
- Parquet/ORC ファイル形式に完全対応

### 主な対応範囲

- **Amazon Redshift Serverless**: 完全対応
- **プロビジョニング型データウェアハウス**: 完全対応
- **リージョン**: AWS 全商用リージョン + GovCloud

---

<!-- _class: section -->

# 1. 背景・課題

取り込みボトルネック解消への取り組み

---

## 背景・課題

### データウェアハウスの課題

- **リアルタイム分析**: ピーク需要時のデータ取り込み遅延
- **継続的な ETL**: 複数の並行ワークロード間での競合
- **高頻度レポーティング**: 時間に敏感なデータ操作での制約

### 従来の同時実行スケーリング

- **対応範囲**: 読み取り（SELECT）クエリのみ
- **制限事項**: 書き込み（COPY）は リソース競合により遅延
- **結果**: 取り込み速度**か**クエリパフォーマンスの選択を強いられた

---

## 市場背景

### エンタープライズの要求

- ETL/ELT パイプラインの継続的な高速化
- リアルタイムデータウェアハウスの人気上昇
- インフラ管理の自動化・サーバレス化の浸透

### AWS の進化

- 2026年3月: Auto-Copy と Zero-ETL で同時実行スケーリング対応開始
- 2026年5月: **COPY クエリ**への直接対応を実現（今回発表）

# 2. 新機能・変更内容

同時実行スケーリングの拡張

---

## 新機能の詳細

### 1. COPY クエリの同時実行スケーリング

**対応ファイル形式**
- Parquet ✅
- ORC ✅
- その他データベース形式: 別途確認

**動作**
- Amazon S3 からの複数ファイルを**並行ロード**
- **キューイング遅延ゼロ**
- 重い並行ワークロードでも対応

---

## 新機能の詳細（続き）

### 2. 自動スケーリング機能

**Amazon Redshift Serverless**
- 需要に応じて**自動的に有効化**
- 需要が減ると**自動的に無効化**
- 手動操作不要

**プロビジョニング型データウェアハウス**
- 事前に同時実行スケーリング設定で制御
- **手動クラスタサイズ変更不要**
- **ワークロードスケジューリング不要**

---

## 実装の簡単さ

### ゼロマイグレーション

```
✅ 既存クエリの変更不要
✅ 設定変更不要
✅ 同時実行スケーリングを有効にするだけ
✅ 即座に効果を得られる
```

### 対応範囲の広さ

- 全 AWS 商用リージョン
- AWS GovCloud (US) リージョン
- Serverless / Provisioned 両対応

# 4. ユースケース

ビジネスシナリオ別の活用

---

## ユースケース 1: リアルタイム分析

### シナリオ

```
営業成績ダッシュボード
↓
営業データが S3 に到着（毎時間）
↓
Redshift COPY で取り込み
↓
BI ツールで即座にダッシュボード更新
```

### 効果

- 遅延を数秒に削減
- 経営層が最新数字を即座に確認可能
- 素早い意思決定を支援

---

## ユースケース 2: 継続的 ETL

### シナリオ

```
データレイク (S3)
↓
複数形式ファイル（Parquet, ORC）を同時にロード
↓
Redshift で ETL 処理
↓
ダウンストリームシステムへ配信
```

### 効果

- パイプライン全体の処理時間短縮
- スケーリング操作の自動化
- 運用コスト削減

---

## ユースケース 3: 高頻度レポーティング

### シナリオ

```
POS システムデータ（複数店舗）
↓
1時間ごとに S3 に集約
↓
複数ファイルを Redshift にロード
↓
日次・週次・月次レポート自動生成
```

### 効果

- キューイング遅延をゼロに
- 自動化による人的コスト削減
- 無停止でのレポート配信
