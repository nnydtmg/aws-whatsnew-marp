# Amazon VPC IPAM のプール割り当てがタグ付けに対応

Amazon VPC IPAM now supports tags on IPAM pool allocations

**カテゴリ:** What's New
**公開日:** 2026-05-26T13:00:00
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-vpc-ipam-tags/)

このページでは、AWS What's Newで発表された「Amazon VPC IPAM now supports tags on IPAM pool allocations」の内容を日本語で要約し、スライド形式で確認できます。

---

## 要約

Amazon VPC IPAM が IPAM プール割り当てへのタグ付けをサポートするようになり、顧客は AWS の他のリソースと同じタグ付けワークフローを使用して IP アドレス割り当てを整理、管理、およびアクセス制御できるようになりました。この機能により、IAM ポリシーを通じた一元的なガバナンスと、マルチアカウント環境での効率的な IP アドレス範囲の検索が可能になります。

## このアップデートで何が変わったか

- 顧客は、IP アドレス割り当ての作成時にタグを付与することも、既存の割り当てにタグを追加することも可能になりました。

- この機能は、IPAM が利用可能なすべての AWS リージョンで、追加コストなしで利用可能です。

## 対象ユーザー

- ネットワーク管理者は、割り当てを環境ごとにタグ付けし、本番環境のネットワーキングロールのみがプールから割り当てることを許可する IAM ポリシーを適用できるようになりました。

## 詳細

- Amazon VPC IPAM（IP Address Manager）が、IPAM プール割り当てに対するタグ機能をサポートするようになりました。

- 顧客は、IP アドレス割り当ての作成時にタグを付与することも、既存の割り当てにタグを追加することも可能になりました。

- これらのタグは AWS Identity and Access Management（IAM）およびサービスコントロールポリシーで参照でき、大規模な IP アドレス使用状況の一元的なガバナンスが実現します。

- ネットワーク管理者は、割り当てを環境ごとにタグ付けし、本番環境のネットワーキングロールのみがプールから割り当てることを許可する IAM ポリシーを適用できるようになりました。

- 顧客は、すべての IPAM プール全体でタグによって割り当てを検索およびフィルタリングでき、大規模なマルチアカウント環境で特定の IP アドレス範囲を迅速に特定できるようになりました。

- この機能は、IPAM が利用可能なすべての AWS リージョンで、追加コストなしで利用可能です。
