# AWS Capabilities by Region now supports availability notifications

**カテゴリ:** What's New
**公開日:** 2026-05-07
**元記事:** [https://aws.amazon.com/about-aws/whats-new/2026/05/aws-regional-planning-tool-notification](https://aws.amazon.com/about-aws/whats-new/2026/05/aws-regional-planning-tool-notification)

---

## 要約

AWS Capabilities by Regionに新しく可用性通知機能が追加されました。これにより、AWSサービスと機能が対象リージョンで利用可能になった際に、自動的にアラート通知を受け取ることができるようになります。このアップデートは、インフラストラクチャ計画、リージョン拡張準備、およびサービスパリティ監視を行う必要があるソリューションアーキテクトやビルダーにとって、特に有用です。

## 詳細

AWS Capabilities by Region に可用性通知機能が追加されました。これにより、AWSサービスおよび機能が対象リージョンで利用可能になった際に、自動的にアラート通知を受け取ることができます。1,500以上のサービスと機能を37のAWSリージョン全体で追跡することが可能になり、インフラストラクチャ計画とデプロイメント決定が加速されます。AWS Builder Center UIを通じてサービスレベルでサブスクリプション登録でき、選択したリージョン全体の基盤となるすべての機能が自動的にカバーされるため、個別機能の追跡は不要です。通知はAWS Builder Center内のインスタント アラートと、統合された週次メールダイジェストの2つのチャネルで配信されます。通知設定はAWS Builder Centerの「Settings > Notifications」で管理できます。このアップデートは、Builder IDを持つすべての認証済みグローバルユーザーに無料で提供されます。主な利用例としては、特定の機能ローンチの追跡、AWSリージョン間のサービスパリティ監視、および今後のマイグレーションやリージョン拡張への準備が挙げられます。例えば、生成AIアプリケーションを新しいリージョンに拡張するソリューションアーキテクトは、Amazon Bedrockにサブスクライブして、Knowledge BasesやGuardrailsなどの機能が利用可能になると自動更新を受け取ることができます。
