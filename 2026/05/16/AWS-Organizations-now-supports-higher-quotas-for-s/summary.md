# AWS Organizations のサービスコントロールポリシー(SCP)クォータが拡大

AWS Organizations now supports higher quotas for service control policies (SCPs)

**カテゴリ:** What's New
**公開日:** 2026-05-15T15:00:00
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/05/aws-organizations-increased-scp-quotas/)

このページでは、AWS What's Newで発表された「AWS Organizations now supports higher quotas for service control policies (SCPs)」の内容を日本語で要約し、スライド形式で確認できます。

---

## 要約

AWS Organizationsのサービスコントロールポリシー(SCP)の上限が拡大され、ノードあたりの最大添付数が5から10に、ポリシーサイズが5,120から10,240文字に増加しました。このアップデートは、複雑なセキュリティ要件を持つエンタープライズ組織が、より細粒度のアクセス制御と包括的なセキュリティコントロールを実装する際に特に有用です。

## このアップデートで何が変わったか

- AWS Organizationsのサービスコントロールポリシー(SCP)の上限が引き上げられました。

- 単一ノード(ルート、OU、またはアカウント)に添付できるSCPの最大数が5から10に増加しました。

- SCPの最大サイズが5,120文字から10,240文字に増加しました。

- より細粒度のアクセス許可と条件を持つSCPを作成できるようになりました。

- ノードごとにより多くのSCPを添付して、組織全体にわたるより包括的なセキュリティコントロールを構築できます。

- この更新は、すべての商用AWSリージョン、AWS GovCloud(US)リージョン、および中国リージョンで利用可能です。

- すべての組織に自動的に利用可能であり、追加のアクション不要です。
