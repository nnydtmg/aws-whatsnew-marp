# Amazon Redshift、AWS Graviton搭載のRGインスタンスがローンチ

Amazon Redshift launches RG instances powered by AWS Graviton

**カテゴリ:** What's New
**公開日:** 2026-05-12
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-redshift-rg-instances-powered-by-graviton)

このページでは、AWS What's Newで発表された「Amazon Redshift launches RG instances powered by AWS Graviton」の内容を日本語で要約し、スライド形式で確認できます。

---

## 要約

Amazon Redshift RG インスタンスは、AWS Graviton プロセッサを搭載した新世代のノードであり、前世代の RA3 インスタンスと比較して最大 2.4 倍高速なパフォーマンスを vCPU あたり 30% 低い価格で実現するものです。このアップデートは、データウェアハウスおよびデータレイクワークロードを統一されたエンジンで実行し、パフォーマンスとコスト効率の両立を求める組織に最適です。

## このアップデートで何が変わったか

### 1. 新しいインスタンスファミリーの導入

- Amazon Redshift RG インスタンスは、AWS Graviton プロセッサを搭載した新世代のプロビジョニングクラスタノードです
- RA3 インスタンスの後継として位置づけられています
- rg.xlarge と rg.4xlarge の 2 つのインスタンスサイズで利用可能

### 2. 大幅なパフォーマンス向上

- RG インスタンスは、前世代の RA3 インスタンスと比較して最大 2.4 倍高速なパフォーマンスを実現
- データウェアハウスワークロードでは RA3 比で最大 2.2 倍
- Apache Iceberg クエリでは最大 2.4 倍
- Parquet ワークロードでは最大 1.5 倍高速

### 3. 優れた費用対効果

- vCPU あたりの価格が 30% 低くなっており、コスト効率が向上
- パフォーマンス向上とコスト削減を同時に実現
- 実行時間の短縮により消費電力も削減

### 4. 統合データレイククエリエンジン

- Redshift のカスタムビルド ベクトル化データレイククエリエンジンが搭載
- Apache Iceberg および Parquet データをネイティブ処理可能
- 単一エンジンでデータウェアハウスとデータレイク全体の SQL 分析を実行可能

### 5. Spectrum 統合による利点

- 単一エンジンの活用により Redshift Spectrum の別スキャンフリートが不要
- テラバイト単位のスキャンに関連する料金が削減
- クエリ管理が単純化

### 6. 最適化技術の強化

- スマートプリフェッチ: データ先読みで遅延を最小化
- NVMe キャッシング: 高速メモリを活用した処理
- ベクトル化 Parquet スキャン: SIMD 命令による並列処理
- ファイルおよびパーティションレベルのプルーニング: 不要なデータアクセス排除
- Just-In-Time (JIT) Analyze: 手動チューニングなしで一貫した高速クエリ実現

### 7. スムーズなマイグレーション

- 既存の RA3 クラスタからの移行が容易
- スナップショット & リストア機能で移行可能
- エラスティックリサイズでダウンタイム最小化
- クラシックリサイズによる段階的な移行にも対応

### 8. 柔軟な価格オプション

- オンデマンド: 使用量ベースの柔軟な価格設定
- 1 年の予約インスタンス: 中期割引
- 3 年の予約インスタンス: 長期割引
- すべての予約インスタンスで前払いなしオプション

### 9. グローバル対応

- 25 以上の AWS リージョンで利用可能
- 世界中のユーザーが直ぐにアクセス可能

### 10. ターゲットユースケース

- データウェアハウスワークロード: 高速分析が必要
- データレイク統合: Apache Iceberg/Parquet データの高速クエリ
- 混合分析環境: ウェアハウス＋レイク統合分析
- パフォーマンス向上とコスト削減の両方を求める企業
- Apache Iceberg および Parquet 形式のデータを扱う組織
- 既存の RA3 ユーザーでアップグレードを検討中の組織

## 詳細

### Amazon Redshift RG インスタンスについて

Amazon Redshift RG インスタンスは、AWS Graviton プロセッサの性能を最大限に活かす設計となっており、従来のインテル系プロセッサと異なるアーキテクチャ最適化が施されています。

### ベクトル化データレイククエリエンジンの特徴

このエンジンは以下の特徴を備えています:

1. **複数フォーマット対応**: Apache Iceberg と Parquet に最適化されたクエリ処理
2. **メモリ効率**: ベクトル化処理により CPU キャッシュの効率的な利用
3. **スキャン最適化**: 不要なデータの自動スキップ機能
4. **自動チューニング**: JIT Analyze による動的最適化

### パフォーマンス向上の背景

複数の技術が組み合わせられることで高速化を実現:

- **ハードウェア**: Graviton プロセッサの優れた計算性能
- **アルゴリズム**: ベクトル化クエリ処理の実装
- **キャッシング**: NVMe キャッシュによる中間データ格納
- **プリフェッチ**: スマートな先読みシステム

### マイグレーション戦略

既存の RA3 ユーザーには複数のマイグレーションオプションが提供されます:

1. **スナップショット & リストア**: 完全なバックアップと復元
2. **エラスティックリサイズ**: ノード数変更による柔軟なスケール変更
3. **クラシックリサイズ**: 段階的なリサイズプロセス

### 価格の利点

vCPU あたり 30% の価格低下により、以下のような効果が期待できます:

- 同じ予算でより多くのコンピュート能力を確保
- クエリ実行時間が短縮されることで消費電力を削減
- 全体的な運用コストの削減
