# AWS MCP ServerがOAuth 2.0に対応

OAuth support for the AWS MCP Server

**カテゴリ:** What's New
**公開日:** 2026-07-09T23:03:00
**元記事:** [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/07/oauth-aws-mcp-server/)

このページでは、AWS What's Newで発表された「OAuth support for the AWS MCP Server」の内容を日本語で要約し、スライド形式で確認できます。

---

## 要約

AWS MCP Serverに対するOAuth 2.0サポートが実装され、AIエージェントが業界標準のOAuthを使用してAWS Sign-Inで直接接続できるようになりました。既存のIAM権限とガバナンスコントロールを活用しながら、対話的および非対話型の認可方法が提供されます。

## このアップデートで何が変わったか

- AWS MCP Serverに対するOAuth対応が新たに追加されました。

- 追加の認証ソフトウェアを必要とせず、既存のAWS ID、サインイン方法、IAM権限、ガバナンスコントロールが引き続き適用されます。

## 対象ユーザー

- 開発者はブラウザを通じた対話的な認可、または非対話型（ヘッドレス）認可をプログラムで実行できます。

## 活用シーン

- このアップデートは、既存のセキュリティインフラストラクチャを活用しながらAIエージェントの統合を簡素化したい組織に適しています。

## 詳細

- AWS MCP Serverに対するOAuth対応が新たに追加されました。

- AIエージェントがAWS Sign-Inを使用して業界標準のOAuthで直接接続できるようになりました。

- 追加の認証ソフトウェアを必要とせず、既存のAWS ID、サインイン方法、IAM権限、ガバナンスコントロールが引き続き適用されます。

- 開発者はブラウザを通じた対話的な認可、または非対話型（ヘッドレス）認可をプログラムで実行できます。

- 管理者はIAMポリシーとOAuth固有の機能（グローバル条件キー、トークン検査・失効API、動的クライアント登録、CloudTrail監査イベント）を使用してOAuthアクセスを管理できます。

- このアップデートは、既存のセキュリティインフラストラクチャを活用しながらAIエージェントの統合を簡素化したい組織に適しています。

- 開発者、システム管理者、セキュリティ管理者にとって有益な機能です。
