# AWS Security Hub が AIインベントリ機能で組織全体のAIアセットを可視化

**発表日: 2026年7月14日**

---

## 概要

### AWS Security Hub AIインベントリの3つの主要特徴

- **自動検出**: Amazon Bedrock、SageMaker などのマネージドAIサービスから自動的にAIリソースを検出
- **包括的な可視化**: 自社開発フレームワーク（Ollama、vLLM など）と外部AI API呼び出しも一元管理
- **脅威との相関**: Amazon GuardDuty の検出結果と連携して、AIアセットのセキュリティリスクを可視化

### 実現される価値

- 複数のAWS環境にわたるAIアセットの統一管理
- AIアセットのセキュリティ脅威と構成ミスを効率的に把握
- 組織全体のAI資産台帳の自動生成

---

## 前提・背景

### 市場の課題

- AI ワークロードの急速な拡大に伴い、AIアセット管理の複雑化が加速
- マネージドサービス、自社開発フレームワーク、外部API の混在により、可視化が困難
- AIアセットとセキュリティ脅威の相関が不十分で、効果的なリスク対応ができない

### AWS Security Hub の進化背景

- 6月2026年: AI Security Best Practices（31個の自動制御）をCSPMで発表
- 現在: AI資産の自動検出と脅威相関機能を統合
- マルチクラウド対応も推進（Azure への拡大）

---

## 主な機能と検出方式

### 1. マネージドAIサービスの検出

- **Amazon Bedrock** インファレンスプロファイル
- **Bedrock AgentCore** エージェント設定
- **Amazon SageMaker** エンドポイント・ノートブック

### 2. 自社開発フレームワークの識別

- Amazon Inspector の **SBOM分析** を活用
- EC2インスタンス上の **Ollama、vLLM、Hugging Face TGI** を特定
- ECRコンテナイメージ内の AI/ML フレームワークを検出

### 3. 外部AI API の監視

- Amazon GuardDuty の **DNS テレメトリ** 活用
- EC2からアクセスされるサードパーティモデルプロバイダーへの呼び出しを追跡

## セキュリティ脅威との相関と分析機能

### AIアセットとインフラストラクチャのマッピング

- 検出されたAIアセットを基盤となるインフラストラクチャに自動マッピング
- Amazon GuardDuty の脅威検出結果と連携
- AWSセキュリティスタック全体の検出結果を統合

### フィルタリング・グループ化・クエリ機能

| 条件 | 説明 |
|------|------|
| **アカウント単位** | 複数アカウント環境での絞り込み |
| **リソースタイプ** | Bedrock、SageMaker など種類別に分類 |
| **検出方法** | マネージド vs 自社開発 vs 外部API |
| **モデルID** | 使用されている具体的なモデルで検索 |

## 効果・メリット

### セキュリティチーム向け

- **一元管理**: 全AIアセットをセキュリティハブで統一監視
- **効率化**: 手動インベントリ作成の廃止
- **迅速対応**: 脅威検出との相関により、インシデント対応が加速

### 運用チーム向け

- **ガバナンス強化**: AIアセットの構成ミスを自動検出
- **コスト把握**: 無駄なAIリソースの特定と削減
- **コンプライアンス**: 組織の AI 使用ポリシーへの準拠を自動チェック

### 経営層向け

- **リスク可視化**: AI投資のセキュリティリスクを定量的に把握
- **意思決定**: データドリブンなAI戦略の策定

---

## 利用可能性・コスト

### 提供範囲

- **セキュリティハブバージョン**: Security Hub v2 で利用可能
- **コスト**: Security Hub Essentials に無料で含まれる
- **追加設定**: 不要（自動有効化）
- **対応リージョン**: AWSのすべてのコマーシャルリージョン

### 前提条件

- AWS Security Hub の有効化
- 対象リソース（Bedrock、SageMaker、EC2 など）のデプロイ

## ユースケース

### ケース1: 中央セキュリティチーム

- 複数事業部門の AI アセットを一箇所で管理
- 組織全体の AI セキュリティリスクを月次レポート化

### ケース2: 大規模組織での一元管理

- マルチアカウント環境での AI 資産台帳の自動生成
- セキュリティスタンダードへの準拠状況を自動監視

### ケース3: AI セキュリティ統制

- AI パイプラインの構成ミスを早期発見
- GuardDuty との連携で、AIアセットへの脅威を即座に把握

## まとめ

### AWS Security Hub AIインベントリの位置付け

- AI ワークロード拡大時代における **必須のセキュリティ基盤**
- マネージド × 自社開発 × 外部 API を **包括的に可視化**
- Security Hub Essentials に無料で提供される **組織全体の AI 資産管理ソリューション**

### 導入効果

1. **即座の効果**: AI資産の自動検出により手動作業を廃止
2. **脅威対応の高速化**: GuardDuty との相関で早期検出・対応
3. **ガバナンス強化**: 構成ミスの自動検出と改善提案

### 次のステップ

1. 現在のAIアセット状況を AIインベントリで把握
2. セキュリティベストプラクティス（6月発表の31制御）を適用
3. 継続的な脅威監視と対応体制の構築
