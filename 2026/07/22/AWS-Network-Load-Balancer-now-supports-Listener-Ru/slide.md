# AWS Network Load Balancerがリスナールールでカスタムトラフィックルーティングに対応

AWS Network Load Balancer now supports Listener Rules for custom traffic routing

**発表日: 2026年7月22日**

---

## 概要

- **リスナールール機能**: 送信元IPアドレスタイプに基づいて接続を異なるターゲットグループへルーティング
- **IPv4/IPv6対応**: IPv4とIPv6の両クライアントに対応する際に、プロトコル変換なしでクライアントIPを保持
- **既存NLBへの追加**: 既存のデュアルスタックNLBに追加でき、再作成は不要
- **全リージョン対応**: すべてのAWSリージョンで追加料金なしで利用可能

---

## 前提・背景

### これまでの課題

- IPv4とIPv6の両クライアントに対応するには、2つの別々のロードバランサーを運用するか、プロトコル変換を採用する必要があった
- プロトコル変換を使用すると、元のクライアントIPアドレスが失われてしまう
- 複数のロードバランサー運用は管理コストが高い
- レイヤー3レベルでのきめ細かいルーティング制御ができなかった

### 市場の動向

- IPv6の普及率が増加し、デュアルスタック対応のニーズが高まっている
- クライアントIPアドレス保持の重要性が増している（セキュリティ、ログ分析など）
- Network Load Balancerの機能拡張により、より多くのユースケースに対応

---

## 主な新機能

### リスナールール機能の特徴

- **IpAddressType条件**: リスナールール内で`IpAddressType`パラメータを使用
- **IPv6トラフィック分離**: IPv6クライアントトラフィックをIPv6ターゲットグループへ自動ルーティング
- **IPv4トラフィック分離**: IPv4クライアントトラフィックをIPv4ターゲットグループへ自動ルーティング
- **クライアントIP保持**: 両方のアドレスファミリーについて、元のクライアントIPアドレスがエンドツーエンドで保持

### 対応プロトコル

- **TCP**: 標準的なTCPリスナーで利用可能
- **UDP**: UDPベースのアプリケーション対応
- **TCP_UDP**: 混合プロトコルリスナー対応
- **TLS**: セキュアな接続で利用可能

---

## 既存NLB機能との連携

### 対応する既存機能

- **接続ドレーニング**: グレースフルなシャットダウンをサポート
- **ターゲットグループスティッキネス**: セッション保持機能と連携
- **クロスゾーン負荷分散**: 複数AZへの分散配置に対応
- **加重ターゲットグループ**: 負荷配分の柔軟な制御が可能
- **クライアントIP保持**: オリジナルのクライアントIP情報を維持

### NLB再作成の不要性

- 既存のデュアルスタックNLBにリスナールールを追加する場合、NLBの再作成が不要
- 既存の設定を保持したまま機能拡張が可能
- ダウンタイムなしでのアップグレードが実現

---

## ユースケース

### 1. マルチプロトコルアプリケーション

- IPv4クライアント→IPv4バックエンド
- IPv6クライアント→IPv6バックエンド
- 1つのロードバランサーで両方のトラフィックを効率的に管理

### 2. グローバルアプリケーション

- IPv6対応地域からのトラフィックを最適化
- クライアント近接のターゲットへのルーティング
- パフォーマンス向上とレイテンシ削減

### 3. レガシーシステム移行

- 既存IPv4システムと新規IPv6システムの共存
- 段階的なIPv6対応が可能
- クライアントIP情報の維持により監査ログも完全

### 4. セキュリティアプライアンス

- クライアント情報の保持でセキュリティ監視を強化
- DDoS対策での詳細なトラフィック分析
- ジオロケーション分析が正確に実施可能

---

## 利用方法

### 設定ステップ

1. **デュアルスタックNLBの確認**: 既存のIPv4/IPv6対応NLBを確認
2. **ターゲットグループの準備**: IPv4用とIPv6用のターゲットグループを事前に作成
3. **リスナールール追加**: NLBリスナーに新しいリスナールールを追加
4. **IpAddressType条件の設定**: `ipv6`または`ipv4`のいずれかを指定
5. **ターゲットグループをマッピング**: 対応するターゲットグループにルーティング

### 料金

- **追加費用**: なし
- **適用される料金**: 標準的なNLB料金のみ
  - ロードバランサー時間単価
  - LCU (Load Balancer Capacity Unit) 単価

## まとめ

### 主な利点

- ✅ **管理の簡素化**: 1つのNLBで IPv4/IPv6を効率的に管理
- ✅ **コスト削減**: 複数ロードバランサーの不要化
- ✅ **ユーザー体験向上**: クライアントIP保持でセキュリティ強化
- ✅ **グローバル展開**: IPv6対応地域への最適なサービス提供
- ✅ **ダウンタイムなし**: 既存NLBへの追加で無中断アップグレード
